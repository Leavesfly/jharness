# 09 - 会话与记忆

> 位于 `io.leavesfly.jharness.session` 与 `io.leavesfly.jharness.core.MemoryManager`。本章讲解会话快照持久化、跨会话记忆、桥接会话三大子系统。

## 1. 组件总览

| 组件 | 包 | 核心类 |
|------|----|--------|
| **会话快照** | `session.sessions` | `SessionStorage`、`SessionSnapshot` |
| **跨会话记忆** | `core` | `MemoryManager`（含内部类 `MemoryEntry` / `MemoryStats`） |
| **桥接会话** | `session.bridge` | `BridgeSessionManager`、`WorkSecretHelper` |

## 2. SessionSnapshot：会话快照

`SessionSnapshot` 是一条会话的完整不可变载体（all-final 字段 + `@JsonCreator` 构造）：

| 字段 | 类型 | 说明 |
|------|------|-----|
| `sessionId` | `String` | 唯一标识（`session-{id}.json` 文件名锚点） |
| `cwd` | `String` | 快照拍摄时的工作目录 |
| `model` | `String` | 使用的 LLM 模型 |
| `messages` | `List<ConversationMessage>` | 完整对话历史（含 text / image / tool_use / tool_result 内容块） |
| `usage` | `UsageSnapshot` | token 使用量（输入 / 输出 / cache） |
| `createdAt` | `Instant` | ISO-8601 时间戳 |
| `summary` | `String` | 摘要（可为 null） |
| `messageCount` | `int` | 消息条数（冗余，用于列表快速展示） |

所有字段均有 `@JsonProperty`（snake_case 映射），可与历史 OpenHarness 项目的文件格式互通。

## 3. SessionStorage：持久化

### 3.1 存储路径

默认 `~/.openharness/sessions/`（在 `CommandRegistry#registerAllCommands` 中显式拼接，为兼容历史数据保留 `openharness` 前缀；新项目可自行改向 `.jharness/sessions/`）。

文件命名：`session-{sessionId}.json`

### 3.2 核心 API

```java
SessionStorage storage = new SessionStorage(sessionsDir);

storage.saveSession(SessionSnapshot snapshot);       // 保存
SessionSnapshot snap = storage.loadSession(id);       // 加载（返回 null 表示不存在或失败）
List<SessionSnapshot> recent = storage.listSessions(limit);   // 按 createdAt 倒序
```

### 3.3 并发与原子性

```java
private final Map<String, Object> saveLocks = new ConcurrentHashMap<>();
```

- **Per-session 锁**：同一 `sessionId` 串行写，不同会话并行写；
- **写入完成后移除锁对象**，防止 `saveLocks` 无限增长导致内存泄漏；
- **临时文件 + 原子移动**：先写 `session-{id}.tmp`，再 `Files.move(..., ATOMIC_MOVE, REPLACE_EXISTING)`，避免写入中断造成文件损坏。

### 3.4 路径遍历防御

`loadSession(id)` 对 `sessionId` 做严格校验：

```java
// 1. 非空
if (sessionId.isBlank()) return null;
// 2. 拒绝任何路径分隔符与相对路径片段
if (sessionId.contains("/") || sessionId.contains("\\") || sessionId.contains("..")) return null;
// 3. normalize 后二次验证仍在 sessionsDir 内
if (!file.startsWith(sessionsDir.normalize())) return null;
```

这样可阻挡 `sessionId="../etc/passwd"` 之类的攻击。

### 3.5 列表策略

`listSessions(limit)` 使用 `Stream` + `Files.list`：
- 过滤 `.json`
- 每条独立 try-catch，单条损坏不影响其他
- `Comparator.comparing(SessionSnapshot::getCreatedAt).reversed()` 按时间倒序
- `.limit(limit)` 控制返回条数

## 4. SessionCommandHandler 提供的命令

注册自 `CommandRegistry#registerAllCommands`（见 [07](07-斜杠命令系统.md)）：

| 命令 | 行为 |
|------|------|
| `/resume` | `storage.loadSession(id)` → `queryEngine.loadMessages` 恢复历史 |
| `/export` | 序列化当前会话为 JSON / Markdown |
| `/share` | 生成可分享快照（基于 `SessionStorage`） |
| `/session` | 查看当前 session 元信息 |
| `/tag` | 给会话打标签 |
| `/rewind` | 回滚 N 轮（修改 `QueryEngine.messages`） |
| `/copy` | 复制最后一条 assistant 消息 |
| `/compact` | 手动调用 `MessageCompactionService` |
| `/context` | 查看当前上下文块 |
| `/summary` | 让 LLM 生成摘要 |

## 5. MemoryManager：跨会话记忆

独立于会话的长期知识存储，典型场景：记住某项目的编码规范、常用命令、已解决问题等。

### 5.1 存储结构

```
~/.openharness/memory/          # 由 CommandRegistry 选用的路径
└── {project}/                  # 项目子目录（sanitize 后的名字）
    ├── {title}.json            # 单条记忆
    ├── {title2}.json
    └── ...
```

### 5.2 `MemoryEntry` 结构

```java
class MemoryEntry {
    String title;
    String content;
    String category;             // 默认 "general"
    LocalDateTime createdAt;
    LocalDateTime updatedAt;
}
```

### 5.3 核心 API

```java
MemoryManager mm = new MemoryManager(memoryDir);

mm.addMemory(project, title, content);                  // category 默认 general
mm.addMemory(project, title, content, category);
String body = mm.readMemory(project, title);
boolean ok   = mm.updateMemory(project, title, content);
boolean ok2  = mm.removeMemory(project, title);
List<String> list    = mm.listMemories(project);
List<String> byCat   = mm.searchByCategory(project, category);
List<String> found   = mm.searchMemories(project, keyword);
MemoryEntry detail   = mm.getMemoryDetails(project, title);

mm.clearProjectMemories(project);                       // 清空某项目
MemoryStats stats = mm.getStats(project);               // 统计
int removed = mm.cleanupOldMemories(project, daysOld);  // 清理 N 天前
```

### 5.4 路径安全

```java
Path projectDir = memoryDir.resolve(sanitize(project)).normalize();
if (!projectDir.startsWith(memoryDir.normalize())) {
    logger.warn("非法的项目路径: {}", project);
    return;
}
```

`sanitize` 去除非法字符并规范化，`.normalize()` + `startsWith` 防穿越。

### 5.5 `MemoryStats` 字段

```java
class MemoryStats {
    int totalCount;
    long totalSize;                       // 字节累计
    Map<String,Integer> categories;       // category → 条数
}
```

由 `/memory` 命令查询（见 `SystemCommandHandler.createMemoryCommand`）。

## 6. BridgeSessionManager：桥接会话

支持把 JHarness 作为 **子会话** 被外部服务调度（远程运行、WebSocket ingress 等）。本身是**纯数据结构 + WorkSecret 编解码**，不包含进程管理逻辑，真正的进程管理由调用方完成。

### 6.1 核心数据类型

| 类 | 字段 |
|---|------|
| `BridgeSessionRecord` | `sessionId`、`command`、`cwd`、`pid`、`status`、`startedAt`、`outputPath` |
| `WorkData` | `type`（`SESSION`/`HEALTHCHECK`）、`id` |
| `WorkSecret` | `version`、`sessionIngressToken`、`apiBaseUrl` |
| `BridgeConfig` | `dir`、`machineName`、`maxSessions`、`verbose`、`sessionTimeoutMs` |

默认超时：`DEFAULT_SESSION_TIMEOUT_MS = 24h`。

### 6.2 WorkSecretHelper

提供 `base64url(JSON)` 编解码与 WebSocket URL 构造：

```java
String encoded = WorkSecretHelper.encodeWorkSecret(workSecret);   // → base64url(JSON)
WorkSecret ws  = WorkSecretHelper.decodeWorkSecret(encoded);       // 版本校验 + 字段非空校验
String wsUrl   = WorkSecretHelper.buildSdkUrl(apiBaseUrl, sessionId);
//  → wss://host/v1/session_ingress/ws/{sessionId}
//  或 ws://localhost/v2/session_ingress/ws/{sessionId}  （本地调试）
```

关键实现细节：
- **显式 UTF-8**：`json.getBytes(StandardCharsets.UTF_8)`，避免跨平台 base64 字节序差异
- **无需手动补 padding**：`Base64.getUrlDecoder()` 自动处理
- **版本校验**：仅支持 `version == 1`
- **必填字段**：`sessionIngressToken`、`apiBaseUrl` 缺一不可

### 6.3 `/bridge` 命令

由 `BridgeCommandHandler.createBridgeCommand` 注册，提供配置查看 / 编码生成 / 健康检查等子命令。

## 7. 配合 QueryEngine 的使用流程

```
// 交互开始：创建 / 恢复
SessionSnapshot snap = storage.loadSession(id);
if (snap != null) {
    engine.loadMessages(snap.getMessages());
    engine.setCwd(Paths.get(snap.getCwd()));
}

// 交互进行中：定期保存
SessionSnapshot s = new SessionSnapshot(
    id, engine.getCwd().toString(), model,
    engine.getMessages(),
    engine.getCostTracker().toUsageSnapshot(),
    Instant.now(),
    summary, engine.getMessages().size());
storage.saveSession(s);

// 交互结束：最终保存
storage.saveSession(buildSnapshot(...));
```
