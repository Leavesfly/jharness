# 16 - 配置管理

> 涉及 `io.leavesfly.jharness.core.config`、`io.leavesfly.jharness.core.state`、`io.leavesfly.jharness.prompts`。本章覆盖静态配置、运行时状态、提示词装配三大主题。

## 1. 配置来源优先级

从低到高（后者覆盖前者）：

1. **代码内默认值**（`Settings.defaults()`）
2. **内置 resources**（classpath 下的 `resources/defaults/*.json`）
3. **用户配置**（`~/.jharness/settings.json`）
4. **项目配置**（`<cwd>/.jharness/settings.json`）
5. **环境变量**（如 `JHARNESS_MODEL`）
6. **CLI 参数**（`--model`、`--permission-mode` 等）

## 2. 数据目录布局

```
~/.jharness/                      # 用户级全局
├── settings.json                 # 主配置
├── credentials.json              # API Key（或走 OS keychain）
├── sessions/                     # SessionStorage 存储
│   └── session-{id}.json
├── memory/                       # MemoryManager 存储
│   └── {project}/
├── skills/                       # SkillLoader 用户层
│   └── *.md
├── plugins/                      # PluginLoader 用户层
│   └── {plugin-name}/
├── hooks.json                    # 用户 Hook 配置
├── mcp.json                      # 用户 MCP 配置
├── output_styles/                # OutputStyleLoader 用户层
│   └── *.md
├── teams/                        # TeamRegistry 存储
│   └── team-{id}.json
└── cron_jobs.json                # CronRegistry 存储

<cwd>/.jharness/                  # 项目级（随仓库分发）
├── settings.json
├── skills/
├── plugins/
├── hooks.json
└── mcp.json
```

> 部分子系统（Session / Memory）沿用 `~/.openharness/` 兼容老数据，详见 [09](09-会话与记忆.md)。

## 3. `Settings`：静态配置模型

结构大致如下（按实际字段以 `Settings.java` 为准）：

```java
public class Settings {
    String model;
    String baseUrl;
    String apiKey;                       // 敏感字段，优先走 env/keychain
    String permissionMode;               // "default" / "fullAuto" / "plan"
    List<String> allowedTools;
    List<String> deniedTools;
    List<String> deniedCommands;
    List<PathRule> pathRules;
    Map<String,Boolean> enabledPlugins;  // 插件启用开关
    String theme;
    boolean vimEnabled;
    boolean voiceEnabled;
    String outputStyle;
    // ...
}
```

加载流程：
```
SettingsLoader.load(cwd)
  ├─ Settings base = Settings.defaults()
  ├─ mergeFromFile(~/.jharness/settings.json)
  ├─ mergeFromFile(<cwd>/.jharness/settings.json)
  ├─ mergeFromEnv(System.getenv())
  └─ return base
```

合并策略：简单字段后覆盖前；`List` 字段追加；`Map` 字段合并 key。

## 4. `AppState`：运行时状态模型

对比 `Settings`：`Settings` 是**启动时的快照**；`AppState` 是**正在变化的实时状态**。

### 4.1 字段（节选）

```java
public class AppState {
    String model;
    String permissionMode;
    String theme;
    String cwd;
    String provider;
    String baseUrl;
    boolean vimEnabled;
    boolean voiceEnabled;
    String statusMessage;
    String outputStyle;
    List<String> activeSkills;
    int totalInputTokens;
    int totalOutputTokens;
    double totalCost;
    int messageCount;
    // ...（共 20+ 字段）
}
```

UI / 其他组件通过 `AppStateStore` 订阅变化。

### 4.2 `AppStateStore`：可观察存储

```java
public class AppStateStore {
    private volatile AppState current;
    private final List<BiConsumer<AppState, AppState>> listeners;

    public AppState getState();
    public void setState(AppState newState);
    public Subscription subscribe(BiConsumer<AppState, AppState> listener);
    private void notifyListeners(AppState oldS, AppState newS);
}
```

关键特性：
- `volatile current` 保证读一致性
- 每次 `setState` 触发 `notifyListeners(old, new)`
- `Subscription` 是一次性 token，调用 `unsubscribe` 安全移除
- 监听器抛异常不会影响其他监听器（try/catch 包裹）

### 4.3 典型使用

```java
// 初始化（在 JHarnessApplication）
AppStateStore store = new AppStateStore(AppState.fromSettings(settings));

// UI 订阅
store.subscribe((oldS, newS) -> {
    statusBar.setModel(newS.getModel());
    statusBar.setPermissionMode(newS.getPermissionMode());
});

// CostTracker 更新 token
store.setState(store.getState().withTokenUsage(inTok, outTok, cost));

// 权限模式变更（/permissions 命令）
store.setState(store.getState().withPermissionMode("plan"));
```

## 5. 提示词装配

### 5.1 `SystemPromptBuilder`

```java
SystemPromptBuilder builder = new SystemPromptBuilder();
String systemPrompt = builder
    .withBaseInstructions()              // 读取内置 base prompt
    .withEnvironmentInfo(env)            // 注入 cwd、OS、shell 等
    .withSkills(skillRegistry)           // 追加 "## 可用技能" 段
    .withOutputStyle(outputStyle)        // 追加输出风格指引
    .withClaudeMd(claudeMdContent)       // 追加项目 CLAUDE.md
    .build();
```

装配顺序自上而下拼接为一个 `String`，每段用 `\n\n` 分隔，`QueryEngine.setSystemPrompt` 使用。

### 5.2 `ClaudeMdLoader`：项目说明文件

兼容 Claude Code 生态：自动查找 `CLAUDE.md` / `AGENTS.md`，从当前 `cwd` 向上最多 **32 层** 查找，支持符号链接检测避免循环。

```java
Optional<String> content = ClaudeMdLoader.load(cwd);
```

合并规则：
- 多层同名文件：**由外层到内层拼接**（项目根 → 子目录）
- 子目录优先生效，但外层仍保留
- 符号链接使用 `Files.readSymbolicLink` 检测，已访问过的 real path 不再处理

### 5.3 `OutputStyle` 与 `OutputStyleLoader`

```java
public class OutputStyle {
    String name;
    String content;
    String source;   // "bundled" / "user"
}
```

加载源：
1. 内置：classpath `resources/output_styles/*.md`
2. 用户：`~/.jharness/output_styles/*.md`

用户可新增自己的 `~/.jharness/output_styles/concise.md`：

```markdown
---
name: concise
---
请尽可能简短地回答，不超过 3 段。
```

通过 `/output-style concise` 切换。

## 6. `JacksonUtils`：统一 JSON 配置

```java
public final class JacksonUtils {
    public static final ObjectMapper MAPPER;           // 紧凑输出
    public static final ObjectMapper PRETTY_MAPPER;    // 美化输出（给 settings.json 用）

    private static ObjectMapper buildDefault() {
        return JsonMapper.builder()
            .addModule(new JavaTimeModule())           // Instant / LocalDateTime
            .addModule(new Jdk8Module())               // Optional 等
            .disable(FAIL_ON_UNKNOWN_PROPERTIES)       // 向后兼容：新增字段不崩
            .disable(WRITE_DATES_AS_TIMESTAMPS)        // ISO-8601 字符串
            .serializationInclusion(NON_NULL)          // 不序列化 null
            .deactivateDefaultTyping()                 // 防多态反序列化 RCE
            .build();
    }
}
```

所有涉及配置读写的子系统（SessionStorage、CronRegistry、SkillLoader、PluginLoader、HookLoader、McpClientManager、TeamRegistry、BackendHost 协议等）**必须**使用这两个 Mapper，避免各自 new `ObjectMapper` 带来的行为不一致与 RCE 风险。

## 7. 环境变量约定

| 变量 | 含义 |
|------|------|
| `JHARNESS_MODEL` | 覆盖 `Settings.model` |
| `JHARNESS_BASE_URL` | 覆盖 `baseUrl` |
| `JHARNESS_API_KEY` | API Key（不建议写 settings.json） |
| `DASHSCOPE_API_KEY` / `DEEPSEEK_API_KEY` 等 | Provider 专属 Key |
| `JHARNESS_PERMISSION_MODE` | 覆盖权限模式 |
| `JHARNESS_DATA_DIR` | 自定义数据目录根（覆盖 `~/.jharness`） |
| `JHARNESS_HOOK_DEPTH` | Hook 递归深度（内部传递，用户一般勿设） |
| `OPENHARNESS_HOOK_PAYLOAD` / `JHARNESS_ARGUMENTS` | Hook 内部 payload（子进程读取） |
| `OPENHARNESS_HOOK_EVENT` | Hook 事件名（子进程读取） |

## 8. 敏感信息处理

### 8.1 API Key

优先级：
1. CLI `--api-key <v>`（**仅开发用，会入 history**）
2. 环境变量
3. `credentials.json`（权限 `chmod 600`，仅 owner 可读）
4. `settings.json`（不推荐，易被 git commit）

`credentials.json` 结构：
```json
{
  "providers": {
    "dashscope":  { "apiKey": "sk-..." },
    "deepseek":   { "apiKey": "sk-..." }
  }
}
```

### 8.2 日志脱敏

JHarness 在写 audit log 与请求日志时对 `api_key`、`authorization`、`token` 等字段默认 **`****` 打码**。用户自定义的敏感字段可通过 `settings.json` 的 `redactKeys` 扩展。

## 9. `/config`、`/settings`、`/model` 等命令

- `/config` — 总览
- `/settings edit` — 打开编辑器修改 `settings.json`
- `/model <name>` — 运行时切换模型（改 `AppState`）
- `/permissions` — 查看 / 调整权限（改 `PermissionChecker` + `AppState`）
- `/theme` — 切换主题
- `/output-style <name>` — 切换输出风格
- `/provider <name>` — 切换 LlmProvider

所有运行时修改都通过 **AppStateStore** 传播到 UI、StatusBar、Backend 等所有订阅者，**无需重启**。

## 10. 配置的保存策略

| 修改源 | 写回 |
|--------|------|
| `/settings edit` | 改 `settings.json` |
| `/model`、`/theme` 等 | 仅改 `AppState`（不落盘） |
| `/model <name> --save` | 同时落到 `settings.json` |
| `AppState.totalCost` 等统计字段 | 不落盘 |

理念：**配置是持久选择，状态是瞬时快照**——两者分离让"临时尝试一个模型"与"永久换默认模型"有不同路径。
