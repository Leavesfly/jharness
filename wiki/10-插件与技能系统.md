# 10 · 插件与技能系统

> 包路径：`io.leavesfly.jharness.extension`
>
> 两套机制：**插件（Plugin）** = 完整功能包；**技能（Skill）** = 可复用的 Markdown 指令模板。

---

## 1. 概念对比

| | 技能（Skill） | 插件（Plugin） |
|---|---------------|----------------|
| **目的** | 把高频 prompt / 工作流封装复用 | 整体扩展 Agent 能力 |
| **格式** | 单个 Markdown 文件 + YAML frontmatter | 目录 + `plugin.json` 清单 |
| **包含** | 名称 + 描述 + 内容 | 技能 / 命令 / 子代理 / Hook / MCP 服务器 |
| **加载** | `SkillLoader` 扫描固定目录 | `PluginLoader` 扫描固定目录 |
| **调用** | `skill` 工具 / `/skills` 命令 / 插件 slash 命令 | 启动时自动激活 |
| **生态兼容** | — | 兼容 Claude Code 的 plugin 目录结构 |

## 2. 技能（Skill）

### 2.1 数据模型

```java
public class SkillDefinition {
    String name;          // 唯一名称
    String description;   // 一行描述
    String content;       // Markdown 正文（prompt 模板）
    String source;        // 来源标记：bundled / user / project / plugin
}
```

### 2.2 文件格式

```markdown
---
name: code-review
description: 系统化代码审查清单
---

# 代码审查清单

请按以下维度审查 $ARGUMENTS：

1. **可读性**：命名 / 注释 / 结构
2. **正确性**：边界 / 异常 / 并发
3. **性能**：算法复杂度 / IO / 内存
4. **安全**：注入 / 越权 / 敏感数据
5. **测试**：单测覆盖 / 关键路径
```

### 2.3 加载顺序

`SkillLoader.loadSkillRegistry(cwd)`：

| 优先级 | 目录 | 来源标记 |
|--------|------|----------|
| 1 | classpath `/skills/*.md`（内置） | `bundled` |
| 2 | `~/.jharness/skills/*.md`（用户） | `user` |
| 3 | `<cwd>/.jharness/skills/*.md`（项目） | `project` |

同名后注册覆盖前注册（`ConcurrentHashMap.put`），方便项目 / 用户覆写默认。

### 2.4 API

```java
SkillRegistry reg = SkillRegistry.withProject(cwd);

SkillDefinition skill = reg.get("code-review");
reg.getSkillNames();      // 全部技能名
reg.getAllSkills();        // 全部 SkillDefinition
reg.register(custom);      // 运行时注册
```

### 2.5 调用方式

#### 方式一：`skill` 工具（LLM 调用）

```json
{
  "tool_calls": [{ "function": {
    "name": "skill",
    "arguments": "{\"name\":\"code-review\",\"args\":\"src/Main.java\"}"
  }}]
}
```

`SkillTool` 读取 `SkillDefinition.content` → 占位符替换 → 把渲染结果作为 `ToolResult.content` 返回。LLM 在下一轮以"用户消息"形式接收。

#### 方式二：`/skills` 命令（用户调用）

```
/skills                列出全部技能
/skills enable <name>  启用（仅记录状态）
/skills show <name>    查看正文
```

#### 方式三：插件 slash 命令

详见 § 4。

## 3. 插件（Plugin）

### 3.1 目录结构

```
my-plugin/
├── plugin.json              # 必需：清单文件
├── skills/                  # 可选：技能 .md 列表
│   └── *.md
├── commands/                # 可选：slash 命令模板
│   └── *.md
├── agents/                  # 可选：子代理定义
│   └── *.md
├── hooks.json               # 可选：Hook 注册（与 manifest.hooksFile 关联）
└── mcp-servers.json         # 可选：MCP 服务器配置（与 manifest.mcpFile 关联）
```

### 3.2 `plugin.json`（PluginManifest）

```json
{
  "name": "my-plugin",
  "version": "1.0.0",
  "description": "为 Agent 增加 X 能力",
  "enabledByDefault": true,

  "author": "Alice",
  "homepage": "https://example.com/my-plugin",
  "repository": "https://github.com/alice/my-plugin",
  "license": "Apache-2.0",
  "keywords": ["code-review", "java"],

  "skillsDir": "skills",
  "commandsDir": "commands",
  "agentsDir": "agents",
  "hooksFile": "hooks.json",
  "mcpFile": "mcp-servers.json"
}
```

未识别字段会被 `@JsonIgnoreProperties(ignoreUnknown = true)` 忽略，**兼容 Claude Code 插件清单**。

### 3.3 加载路径（含 Claude Code 兼容回退）

`PluginPaths` 提供四级目录：

| 级别 | 路径 | 主 / 兼容 |
|------|------|----------|
| 用户主 | `~/.jharness/plugins/` | 主（启动时自动创建） |
| 用户兼容 | `~/.claude/plugins/` | 回退（不主动创建） |
| 项目主 | `<cwd>/.jharness/plugins/` | 主 |
| 项目兼容 | `<cwd>/.claude/plugins/` | 回退（不主动创建） |

`PluginPaths.listUserPluginRoots()` / `listProjectPluginRoots()` 按优先级返回**已存在**的目录列表。

### 3.4 `LoadedPlugin` 运行时表示

```java
public class LoadedPlugin {
    PluginManifest manifest;
    Path path;
    boolean enabled;
    List<SkillDefinition> skills;            // skills/
    List<SkillDefinition> commandPrompts;    // commands/
    List<SkillDefinition> agentDefs;         // agents/
    Map<String, List<Object>> hooks;          // hooks.json 解析
    Map<String, Object> mcpServers;           // mcp-servers.json 解析
}
```

`commandPrompts` / `agentDefs` 都复用 `SkillDefinition` 作为"Markdown + metadata"的容器，避免重复定义 DTO。

### 3.5 `PluginLoader` 流程

```java
PluginLoader loader = new PluginLoader();
List<LoadedPlugin> plugins = loader.loadAll(cwd);
```

内部逻辑：

1. 遍历 `listUserPluginRoots() + listProjectPluginRoots(cwd)`
2. 每个子目录读取 `plugin.json` → `PluginManifest`
3. `manifest.enabledByDefault` 决定初始 `enabled` 状态
4. 按 `skillsDir` / `commandsDir` / `agentsDir` 路径扫描子目录，逐个解析 Markdown
5. 按 `hooksFile` / `mcpFile` 加载 JSON 配置
6. 同名插件：项目级覆盖用户级（按加载顺序后注册覆盖）

### 3.6 注入到 JHarness

`QueryEngineBuilder` 装配时：

```java
List<LoadedPlugin> plugins = pluginLoader.loadAll(cwd);

for (LoadedPlugin p : plugins) {
    if (!p.isEnabled()) continue;

    // 技能 → SkillRegistry
    p.getSkills().forEach(skillRegistry::register);

    // slash 命令 → CommandRegistry
    p.getCommandPrompts().forEach(def ->
        commandRegistry.register(PluginSlashCommand.fromSkill(def, p.getName())));

    // Hook → HookRegistry
    p.getHooks().forEach((event, list) -> {
        HookEvent ev = HookEvent.valueOf(event.toUpperCase());
        list.forEach(def -> hookRegistry.register(ev, def));
    });

    // MCP 服务器 → McpClientManager
    p.getMcpServers().forEach(mcpManager::addServer);

    // 子代理 → 在 agent_spawn 工具内按名查找
}
```

## 4. 插件命令（PluginSlashCommand）

### 4.1 概览

把 `commands/<name>.md` 的 Markdown 正文作为 prompt 模板，用户调用 `/<name> args...` 时：

1. 取出 prompt 模板
2. 按占位符规则注入 args
3. 把渲染结果作为一次新的 user 消息提交给 `QueryEngine`
4. LLM 处理后产生回复

### 4.2 占位符（对齐 Claude Code）

| 占位符 | 替换为 |
|--------|--------|
| `$ARGUMENTS` | 全部 args 用空格连接 |
| `{{args}}` | 同上 |
| `$1`, `$2`, ..., `$99` | 第 i 个 arg（越界为空串） |

实现：`$1` 用正则一次性替换，避免"`$1` 先于 `$10` 被替换"导致字符串腐蚀。

### 4.3 示例

`~/.jharness/plugins/code-tools/commands/explain.md`：

```markdown
---
name: explain
description: 解释一段代码
---

请用通俗易懂的中文解释下面这段代码的功能：

```$1
$2
```

注意覆盖：1. 输入输出 2. 关键算法 3. 潜在边界。
```

用户调用：

```
/explain java "for (int i=0; i<arr.length; i++) System.out.println(arr[i]);"
```

渲染后作为 user 消息提交。

## 6. 插件信任系统

`extension/plugins/trust/TrustStore.java` 实现**基于 author 的信任链**：

- **持久化位置**：`~/.jharness/trusted_authors.json`，格式为 JSON 字符串数组：`["alice", "bob", "anthropic"]`
- **信任粒度**：按 `manifest.author` 字段；同一 author 的所有插件共享信任状态
- **API**：`trust(author)` / `revoke(author)` / `isTrusted(author)` / `list()` / `evaluate(author, TrustPolicy)`
- **线程安全**：所有写操作走内部 `synchronized(lock)`，读取返回快照

`TrustPolicy` 枚举（由调用方在 install 时指定）：

| 策略 | 未知 author 行为 |
|------|----------------|
| `STRICT` | 直接拒绝安装 |
| `WARN` | 记 warn 日志，放行（默认） |
| `AUTO` | 自动加入信任列表（仅用于脚本化批量安装） |

`manifest.author` 为空时同样按上述策略处理（STRICT 拒绝，WARN/AUTO 放行）。

`extension/plugins/marketplace/` 子包目前**未在本章详述**，相关使用方式以源码为准。

## 8. 编写一个完整插件

`~/.jharness/plugins/my-plugin/plugin.json`：

```json
{
  "name": "my-plugin",
  "version": "1.0.0",
  "description": "示例插件：审查 + 解释 + Hook",
  "enabledByDefault": true,
  "skillsDir": "skills",
  "commandsDir": "commands",
  "hooksFile": "hooks.json"
}
```

`skills/code-review.md`：（同 § 2.2 示例）

`commands/review.md`：

```markdown
---
name: review
description: 调用 code-review 技能审查指定文件
---

请使用 code-review 技能审查 $1，args 是 $2。
```

`hooks.json`：

```json
{
  "session_start": [
    { "type": "command", "command": "echo 'Plugin my-plugin loaded'" }
  ],
  "user_prompt_submit": [
    { "type": "prompt", "prompt": "记住：所有代码要遵循公司编码规范。" }
  ]
}
```

启动 JHarness 后：

- `/skills` 列表中应能看到 `code-review`
- `/review src/Main.java "重点关注并发"` 触发自定义 prompt
- 启动时执行 echo
- 每次用户输入前注入"记住..."提示
