# 02 - 快速开始

本文档指引你从零完成 JHarness 的构建、配置与首次运行。

## 1. 环境准备

| 依赖 | 最低版本 | 备注 |
|------|---------|-----|
| JDK | **17** | `pom.xml` 指定 `maven.compiler.source/target=17` |
| Maven | 3.6+ | 支持 Maven Wrapper 亦可 |
| 操作系统 | macOS / Linux / Windows | TUI 在 Unix-like 终端体验最佳 |
| LLM API Key | — | 任一 OpenAI 兼容厂商均可，见下 |

验证环境：

```bash
java -version          # 需 >= 17
mvn -v
```

## 2. 获取源码与构建

```bash
# 克隆仓库
git clone <repo-url> jharness
cd jharness

# 编译 + 单元测试
mvn clean test

# 打包为 fat jar（包含全部依赖）
mvn clean package -DskipTests

# 产物位于
ls target/jharness-0.1.0-SNAPSHOT.jar
```

打包行为由 `maven-shade-plugin` 控制，`MANIFEST.MF` 的 `Main-Class` 会指向 `io.leavesfly.jharness.JHarnessApplication`，可直接 `java -jar` 运行。

## 3. 配置 API Key

JHarness 通过 **环境变量 → 配置文件** 两层查找 Key，**环境变量优先**。

### 方式一：环境变量（推荐）

```bash
# OpenAI 兼容（Qwen / DeepSeek / 本地 vLLM 等）
export OPENAI_API_KEY="sk-xxxxxx"
export OPENAI_BASE_URL="https://dashscope.aliyuncs.com/compatible-mode/v1"

# 或使用 Anthropic 前缀（兼容历史配置）
export ANTHROPIC_API_KEY="sk-xxxxxx"
export ANTHROPIC_BASE_URL="https://dashscope.aliyuncs.com/compatible-mode/v1"

# 覆盖默认模型
export JHARNESS_MODEL="qwen3-max"
export JHARNESS_MAX_TOKENS="8192"
```

支持的环境变量（见 `Settings#loadFromEnvironment`）：

| 环境变量 | 覆盖字段 |
|---------|---------|
| `OPENAI_API_KEY` / `ANTHROPIC_API_KEY` | `apiKey` |
| `OPENAI_BASE_URL` / `ANTHROPIC_BASE_URL` | `baseUrl` |
| `JHARNESS_MODEL` | `model` |
| `JHARNESS_MAX_TOKENS` | `maxTokens` |

### 方式二：配置文件

创建 `~/.jharness/settings.json`：

```json
{
  "model": "qwen3-max",
  "baseUrl": "https://dashscope.aliyuncs.com/compatible-mode/v1",
  "maxTokens": 4096,
  "maxTurns": 8,
  "permissionMode": "DEFAULT",
  "theme": "default",
  "allowedTools": ["read_file", "grep", "glob"],
  "deniedTools": [],
  "pathRules": [
    { "pattern": "src/**", "allow": true },
    { "pattern": "/etc/**", "allow": false }
  ],
  "deniedCommandPatterns": ["rm -rf *", "sudo *"],
  "mcpServers": {},
  "enabledPlugins": {}
}
```

> ⚠️ **安全提示**：`apiKey` 字段由于 `@JsonIgnore` 标注，**不会** 被 `save()` 持久化。推荐始终通过环境变量注入。

## 4. 首次运行

### 4.1 交互式控制台（默认）

```bash
java -jar target/jharness-0.1.0-SNAPSHOT.jar
```

启动后进入交互模式，输入任意自然语言或 `/help` 查看命令：

```
> 帮我总结一下 src/main 目录下有哪些模块
> /status
> /model
> /exit
```

### 4.2 Lanterna TUI 模式

```bash
java -jar target/jharness-0.1.0-SNAPSHOT.jar --tui
```

三区布局：顶部标题栏、中部对话历史、底部输入框 + 状态栏。具体见 [15-TUI 终端界面](15-TUI终端界面.md)。

### 4.3 单次查询（脚本模式）

```bash
java -jar target/jharness-0.1.0-SNAPSHOT.jar \
    -p "列出 pom.xml 中的所有依赖" \
    --max-turns 5
```

退出时自动打印 **📊 用量摘要** 便于成本核算。

### 4.4 权限模式切换

```bash
# 全自动模式（不弹确认，但黑名单/路径规则仍生效）
java -jar .../jharness.jar --permission-mode full_auto

# 计划模式（只读，任何写操作都会被拒绝）
java -jar .../jharness.jar --permission-mode plan
```

## 5. CLI 参数速查

以下选项定义在 `JHarnessApplication`（Picocli `@Option`）：

| 短选项 | 长选项 | 说明 | 默认 |
|-------|--------|-----|------|
| `-p` | `--print` | 单次查询模式，直接输出结果 | — |
| `-m` | `--model` | 指定 LLM 模型 | 取自 `Settings` |
| `-d` | `--debug` | 启用调试日志 | `false` |
| `-c` | `--continue` | 继续上一次会话 | `false` |
| `-r` | `--resume` | 恢复指定 `sessionId` | — |
| — | `--permission-mode` | `default` / `full_auto` / `plan` | `default` |
| — | `--max-turns` | ReAct 循环最大轮次 | `8` |
| — | `--output-format` | `text` / `json` / `stream-json` | `text` |
| — | `--tui` | 启用 Lanterna TUI | `false` |
| `-h` | `--help` | 查看帮助 | — |
| `-V` | `--version` | 版本信息 | — |

## 6. 目录结构约定

| 路径 | 用途 |
|------|------|
| `~/.jharness/settings.json` | 全局配置 |
| `~/.jharness/plugins/` | 用户级插件 |
| `~/.jharness/skills/` | 用户级技能 |
| `~/.openharness/sessions/` | 会话快照（历史命名，仍兼容） |
| `~/.openharness/memory/` | 跨会话记忆 |
| `~/.jharness/data/` | 运行时数据（Cron / 后台任务） |
| `~/.jharness/data/cron_jobs.json` | 持久化 Cron 定义 |
| `<cwd>/.jharness/plugins/` | 项目级插件 |
| `<cwd>/.jharness/skills/` | 项目级技能 |
| `<cwd>/CLAUDE.md` | 项目上下文，会被注入系统提示词 |

## 7. 常用开发命令

```bash
# 运行所有单测
mvn test

# 单跑某个测试类
mvn test -Dtest=JHarnessComprehensiveTest

# Google Java Format 检查
mvn spotless:check

# 自动格式化
mvn spotless:apply
```

## 8. 常见问题

**Q1. 启动时报 `错误: 未配置 API Key`？**

检查 `OPENAI_API_KEY` / `ANTHROPIC_API_KEY` 环境变量是否设置；或在 `~/.jharness/settings.json` 中显式写入 `apiKey`（不推荐）。

**Q2. 调用报 429 / 5xx？**

内置 `RetryPolicy.defaultPolicy()` —— 最多 3 次，初始 1s 指数退避 (×2)。认证失败 (`AuthenticationFailureException`) 不重试直接抛出。

**Q3. TUI 启动后键盘无响应？**

`TerminalUI` 强制使用 `UnixTerminal`，避免 macOS 上 Swing 窗口的 AWT 输入法兼容问题。如仍异常，可通过 `-Djava.awt.headless=true` 启动。

**Q4. 如何让 Agent 仅读不写？**

使用 `--permission-mode plan` 或在交互模式下执行 `/plan`。该模式下所有非只读工具都会被 `PermissionChecker` 拒绝。
