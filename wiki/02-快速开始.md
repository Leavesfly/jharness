# 02 · 快速开始

> 目标：5 分钟内让你在本机跑起 JHarness，并发出第一句对话。

---

## 1. 环境准备

### 必备

- **JDK 17 或更高**（OpenJDK / Oracle JDK / Zulu / Amazon Corretto 均可）

  ```bash
  java -version
  # openjdk version "17.0.x" 或更高
  ```

- **Maven 3.6.3+**

  ```bash
  mvn -v
  ```

### 可选（任选一种 LLM 后端）

| 后端 | 适用 | 准备 |
|------|------|------|
| **本地 Ollama**（推荐） | 离线 / 隐私 / 零成本 | `brew install ollama` → `ollama serve` → `ollama pull qwen3.5:4b` |
| **远端 OpenAI 兼容** | 高性能 / 大模型 | 准备 API Key（OpenAI / DeepSeek / Qwen / Kimi / DashScope 任一） |

---

## 2. 构建

```bash
git clone <repo-url>
cd jharness
mvn clean package
```

产物：`target/jharness-0.1.0-SNAPSHOT.jar`（fat jar，包含全部依赖）。

如遇 `dependencyConvergence` 报错，说明你本机 Maven 缓存里有冲突版本，先 `mvn dependency:tree` 定位，再 `mvn -U` 强制刷新。

---

## 3. 首次启动

### 3.1 本地 Ollama 模式（默认）

```bash
# 1. 启动 Ollama（默认监听 11434）
ollama serve &

# 2. 拉取默认模型
ollama pull qwen3.5:4b

# 3. 启动 JHarness（无需 API Key）
java -jar target/jharness-0.1.0-SNAPSHOT.jar
```

首次启动时，`SettingsBootstrap` 会自动把 classpath 中的 `defaults/settings.json` 释放到 `~/.jharness/settings.json`，并把 `defaults/plugins/` 中的默认插件释放到 `~/.jharness/plugins/`。

随后 `OnboardingService` 会判断是否需要弹出 **Agent Onboarding 向导**：

- **触发条件（必须全部满足）**：交互式模式（未传 `-p`） + 真实终端（`System.console() != null`） + 文本输出格式 + `~/.jharness/.onboarded` 标记文件不存在 + 当前 `apiKey` 为空且 `baseUrl` 非本地端点。
- **本地 Ollama 模式**：因 `baseUrl=http://localhost:11434/v1` 是本地端点，向导会**静默跳过**并直接写入 `.onboarded` 标记。
- **远端 API 模式且未配置任何凭据**：向导会引导你完成 Provider 选择、Base URL、模型名、API Key（脱敏输入）、权限模式四步，结果直接写入 `~/.jharness/settings.json`。
- **跳过向导**：传 `--skip-onboarding`，或在向导第一步选择 `3) 跳过`（会写入 marker，下次不再提示）。
- **重新触发向导**：在交互式会话中执行 `/onboarding`，它会清除 marker；下次启动 JHarness 时自动重新进入向导。

### 3.2 远端 API 模式

```bash
# DeepSeek 示例
export OPENAI_API_KEY=sk-xxx
export OPENAI_BASE_URL=https://api.deepseek.com/v1
export OPENAI_MODEL=deepseek-chat

java -jar target/jharness-0.1.0-SNAPSHOT.jar
```

环境变量等价别名（兼容 Claude Code 用户）：

- `ANTHROPIC_API_KEY` ≡ `OPENAI_API_KEY`
- `ANTHROPIC_BASE_URL` ≡ `OPENAI_BASE_URL`
- `JHARNESS_MODEL` ≡ `OPENAI_MODEL`

## 4. 第一次对话

启动后你会看到类似：

```
JHarness 启动
模型: qwen3.5:4b
权限模式: default
最大轮次: 8
> 你好，帮我列一下当前目录有哪些文件
```

JHarness 会自动调用 `glob` / `bash` 等工具完成任务。**遇到写操作时会询问你是否允许**（这是 `DEFAULT` 权限模式的行为）。

退出：`/exit` 或 `Ctrl+D`。

## 5. 常见运行模式

### 5.1 单次查询（适合脚本化）

```bash
java -jar jharness.jar -p "帮我总结一下 README.md"
```

执行完毕直接退出，输出全文（或 `--output-format json` 输出结构化结果）。

### 5.2 恢复会话

```bash
# 继续上一次会话
java -jar jharness.jar -c

# 恢复指定会话
java -jar jharness.jar -r abc12345
```

会话快照默认保存在 `~/.jharness/data/sessions/<id>.json`。

### 5.3 计划模式（只读）

```bash
java -jar jharness.jar --permission-mode plan
```

适合"先让 AI 出方案，我再决定要不要改"。在此模式下所有写操作（`write_file` / `edit_file` / `bash` 写入 / `multi_edit`）都会被 `ModeDecisionRule` 阻断。

### 5.4 全自动模式

```bash
java -jar jharness.jar --permission-mode full_auto
```

**注意**：黑名单 / 路径规则依然生效，"全自动" ≠ "无安全"。`rm -rf /` 这种危险命令在默认配置中仍会被命令黑名单拦截。

### 5.5 Lanterna TUI

```bash
java -jar jharness.jar --tui
```

适合长时间对话，提供 Markdown 渲染和状态栏。

### 5.6 跳过首次启动向导（CI / 脚本场景）

```bash
java -jar jharness.jar --skip-onboarding
```

或在已有交互式环境中，向导第一步选择 `3) 跳过`。两者都会在 `~/.jharness/.onboarded` 写入标记，防止后续重复弹出。CI/容器场景因为 `System.console() == null`，向导本身也不会触发，加 `--skip-onboarding` 仅用于"语义明确"。

## 5.7 配置加载优先级

JHarness 的配置采用多层合并，从低到高（高优先级覆盖低优先级）：

```
1. 字段硬编码兜底（Settings 构造器初始值）
2. classpath defaults/settings.json（DefaultSettingsLoader 加载）
3. 用户 ~/.jharness/settings.json
4. 环境变量（OPENAI_API_KEY / JHARNESS_MODEL / ...）
5. CLI 选项（-m / --max-turns / --permission-mode / ...）
```

`Settings.load()` 内部按这个顺序逐层 `mergeFromJson` 增量覆盖；环境变量会在文件加载后再叠加一次，保证 `export OPENAI_*` 永远能生效。详见 [`16-配置管理.md`](16-配置管理.md)。

## 7. 持久化目录

JHarness 默认在 `~/.jharness/` 下管理所有持久化数据：

```
~/.jharness/
├── settings.json          # 全局配置（首次启动自动从 classpath defaults/ 生成）
├── .onboarded             # Onboarding 完成标记（由 OnboardingMarker 维护）
├── plugins/               # 用户级插件
├── skills/                # 用户级技能
└── data/
    ├── sessions/          # 会话快照
    ├── memories/          # 跨会话记忆（按 project 子目录组织）
    ├── tasks/             # 后台任务输出
    └── cron_jobs.json     # Cron 定义
```

`.onboarded` 标记文件由 `config.OnboardingMarker` 统一管理：在向导完成 / 用户选择跳过 / 检测到已有可用配置时写入；`/onboarding` 命令会删除它，让下次启动重新进入向导。

如需更换目录，目前需通过修改 `Settings#getDefaultConfigDir/getDefaultDataDir` 实现（后续可能开放 `JHARNESS_HOME` 环境变量）。

## 8. 项目上下文（CLAUDE.md）

在你的项目根目录创建 `CLAUDE.md`，JHarness 会在系统提示词中自动注入它的内容：

```markdown
# 项目说明

这是一个 Spring Boot 微服务。

## 编码规范
- 使用 Google Java Style
- 所有 API 必须编写单元测试

## 关键模块
- `auth/` - JWT 鉴权
- `payment/` - 支付集成（接 Stripe）
```

注入逻辑见 `app.bootstrap.McpBootstrap#loadProjectContext`。

## 9. 故障排查

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| 启动报 `API Key 不能为空` | 既未设置环境变量，也未指向本地端点 | 设置 `OPENAI_API_KEY`，或 `OPENAI_BASE_URL=http://localhost:11434/v1`，或运行 `/onboarding` 重新走向导 |
| `dependencyConvergence` 编译失败 | Maven 缓存有冲突版本 | `mvn -U clean package` |
| Ollama 连不上 | Ollama 未启动 / 端口被占 | `curl http://localhost:11434/api/tags` 验证 |
| 工具调用一直被询问 | `DEFAULT` 模式行为 | 切 `--permission-mode full_auto`，或用 `/permissions` 加白名单 |
| 中文乱码 | 终端编码非 UTF-8 | `export LANG=en_US.UTF-8` |
| SSE 中途断开 | 网络抖动 | `RetryPolicy` 会自动重试 3 次；网络极差可调大 `readTimeoutSeconds` |
| Onboarding 向导没有弹出 | 当前不是真实终端（管道/重定向/IDE Run）或已存在 `.onboarded` 标记 | 删除 `~/.jharness/.onboarded` 或执行 `/onboarding` 后在真实终端重启 |
| Onboarding 向导每次都弹 | 写 marker 时权限不足 | 检查 `~/.jharness/` 目录可写权限；或显式传 `--skip-onboarding` 临时绕过 |

更多诊断：`/doctor` 命令会检查 API Key、Base URL、Java 版本、磁盘空间等环境项。
