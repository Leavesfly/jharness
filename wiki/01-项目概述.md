# 01 - 项目概述

## 🎯 项目定位

**JHarness** 是一个使用 **Java 17** 实现的轻量级 AI Agent 框架，是开源项目 **OpenHarness** 的 Java 端口实现。它将现代 AI Coding Agent 的核心能力以模块化、可扩展的方式封装到 JVM 生态中，面向以下场景：

- **交互式 AI 编码助手** — 通过 CLI / TUI 与大模型对话，完成代码阅读、编辑、调试、运维等任务。
- **Agentic 自动化脚本** — 以非交互式的 `--print` 模式被 CI / 脚本驱动执行一次性任务。
- **企业内集成平台** — 作为 Java 服务嵌入到已有系统，扩展自定义工具、权限、Hook 与 MCP 资源。

## ✨ 核心特性

### 1. 基于 ReAct 循环的查询引擎

`QueryEngine` 实现了标准的 **Reasoning + Acting** 循环：

```
用户 → submitMessage → LLM 流式响应 → 工具调用解析 → 权限校验 → 并行执行工具 → 结果回喂 LLM → …（循环直至完成或达到 maxTurns）
```

支持 **流式 SSE 输出**、**CompletableFuture 并行工具执行**、**消息压缩**、**取消请求**、**预算上限中断**等特性。

### 2. OpenAI 兼容的多模型后端

`OpenAiApiClient` 严格按照 **OpenAI Chat Completions + SSE** 协议实现，可无缝对接：

- 阿里云 DashScope（默认，Qwen 系列）
- OpenAI / Azure OpenAI
- DeepSeek / Moonshot / 智谱
- 本地部署（vLLM、Ollama、LM Studio 等）

内置 **指数退避重试**（`RetryPolicy`）、**认证失败快速失败**、**请求超时可配置**、**HTTP 连接池复用**。

### 3. 40+ 内置工具

工具系统基于抽象基类 `BaseTool<T>`，自动通过反射 + Jakarta Validation 生成 JSON Schema。所有工具经 `ToolRegistry` 统一注册：

| 类别 | 工具（节选） |
|------|-------------|
| 文件操作 | `FileReadTool` / `FileWriteTool` / `FileEditTool` / `MultiEditTool` / `UndoEditTool` / `GlobTool` / `GrepTool` |
| 代码智能 | `LspTool`（符号提取、定义跳转、引用查找、悬停信息） / `NotebookEditTool` |
| 系统交互 | `BashTool` / `SleepTool` |
| 任务编排 | `TodoWriteTool` / `TaskCreateTool` / `TaskGet/List/Stop/Output/UpdateTool` |
| 多智能体 | `AgentTool` / `SendMessageTool` / `TeamCreate/DeleteTool` |
| 网络 | `WebFetchTool` / `WebSearchTool`（Jsoup + DuckDuckGo） |
| MCP | `ListMcpResourcesTool` / `ReadMcpResourceTool` / `McpAuthTool` / `McpToolAdapter`（动态） |
| 定时任务 | `CronCreateTool` / `CronListTool` / `CronDeleteTool` / `RemoteTriggerTool` |
| 模式切换 | `EnterPlanModeTool` / `ExitPlanModeTool` / `EnterWorktreeTool` / `ExitWorktreeTool` |
| 元能力 | `ToolSearchTool` / `BriefTool` / `ConfigTool` / `AskUserQuestionTool` / `SkillTool` |

### 4. 50+ 斜杠命令

`CommandRegistry` 在交互式模式下提供丰富的 `/` 命令，按功能分类：

- **基础**：`/help` `/exit` `/clear` `/status` `/version` `/onboarding`
- **配置**：`/config` `/model` `/permissions` `/theme` `/login` `/logout` `/doctor` `/plan` `/skills` `/tasks` `/mcp`
- **会话**：`/resume` `/export` `/share` `/session` `/tag` `/rewind` `/copy` `/compact` `/context` `/summary`
- **Git**：`/diff` `/branch` `/commit` `/files`
- **系统**：`/memory` `/usage` `/cost` `/stats` `/hooks` `/vim` `/voice` `/output-style` `/keybindings`
- **Cron / Agent**：`/cron` `/agents`
- **扩展**：`/effort` `/passes` `/fast` `/plugin` `/reload-plugins` `/init` `/cd` `/issue` `/pr-comments` `/privacy` `/upgrade` `/release-notes` `/rate-limit` `/bridge` `/feedback`

### 5. 三级权限系统

`PermissionChecker` 实现了严格的安全策略链：

1. 工具黑名单（全模式强制）
2. 命令黑名单（支持 glob + token 归一化，抗 `$IFS` / 引号变形）
3. 路径规则（全模式强制，基于 `realPath` 解符号链接）
4. 工具白名单（仅跳过二次确认）
5. 模式决策：`DEFAULT` / `FULL_AUTO` / `PLAN`

每次 `evaluate` 输出结构化**审计日志**到独立 logger（`jharness.permission.audit`）。

### 6. 扩展三板斧：Plugin / Skill / Hook

- **Plugin**：以目录形式组织（`plugin.json` + `skills/` + `hooks.json` + `mcp.json`），支持用户级 `~/.jharness/plugins/` 与项目级 `<cwd>/.jharness/plugins/` 双层。
- **Skill**：Markdown + YAML Front-matter 定义可复用 AI 行为，三层加载（内置 classpath / 用户 / 项目）。
- **Hook**：在 `SESSION_START/END`、`PRE/POST_TOOL_USE` 四个生命周期点织入 **Command / HTTP / Prompt / Agent** 四类 Hook，内置命令注入防护、SSRF 防护、递归爆炸上限（`MAX_HOOK_DEPTH=3`）。

### 7. MCP 协议原生支持

`McpClientManager` 完整实现 Model Context Protocol 客户端：

- **stdio** 传输（JSON-RPC over 子进程）
- **HTTP** 传输（`followRedirects=false` 防 SSRF）
- 启动时通过 `tools/list` 发现外部工具，动态包装为 `McpToolAdapter` 注册到 `ToolRegistry`
- 与 `PermissionChecker` 注入集成，stdio MCP 服务器 fork 子进程前走权限评估

### 8. 多智能体协调

`AgentOrchestrator` 支持：

- **并行** (`executeParallel`) —— `CompletableFuture.allOf` + 30 min 超时保护
- **顺序** (`executeSequential`) —— 前序结果作为上下文传入后序
- **单任务** (`executeSingle`) —— 独立子引擎

有界线程池（`jharness-agent-N`）+ `CallerRunsPolicy` 回压策略，避免线程爆炸。

### 9. 后台任务与定时调度

- `BackgroundTaskManager`：Shell / Agent 任务的生命周期管理，命名线程 `jharness-bg-task-N`，`AutoCloseable`。
- `CronRegistry`：作业持久化到 `~/.jharness/data/cron_jobs.json`，`ScheduledExecutorService` 驱动，支持 `RemoteTriggerTool` 按需触发。

### 10. 双形态终端界面

- **`ConsoleInteractiveSession`**：纯 ANSI 控制台交互，默认模式。
- **`TerminalUI`（`--tui`）**：基于 **Lanterna 3.1.1** 的全屏 TUI，三区布局（Header / Transcript / Input + StatusBar），支持流式增量渲染。

### 11. 成本与预算管控

`CostTracker` 原子计数 Input/Output/Cache token，按模型价格表（`ModelPricing`）换算 USD，支持：

- 会话累计 / 当日累计（跨日自动重置）
- 每日预算上限，超限抛出 `BudgetExceededException`
- CLI 结束时打印「📊 用量摘要」
