# 01 · 项目概述

> JHarness 是一个 **Java 实现的轻量级 AI Agent 框架**，对标 Anthropic 的 Claude Code / OpenHarness（Python 版本），把同等的核心能力以 Maven 单 jar 的方式带到 JVM 生态。

---

## 1. 项目定位

JHarness 不是又一个对话式 AI 聊天工具，而是一个 **可扩展的 Agent 运行时**：

| 角色 | 它为你做了什么 |
|------|----------------|
| **终端用户** | 一个开箱即用的 AI 编程助手（CLI + TUI 双形态），本地 Ollama 或远端 OpenAI 兼容服务均可 |
| **二次开发者** | 提供 `BaseTool` / `SlashCommand` / `LlmGateway` / `HookRunner` 等清晰扩展点，可在不 fork 主仓的前提下定制业务 Agent |
| **企业平台** | 把 Agent 嵌入 Java 服务（如 SpringBoot 后端），通过 `QueryEngine` API 直接驱动，无需 Python 运行时 |

---

## 2. 与上游 OpenHarness 的关系

JHarness 是 OpenHarness 思想的 Java 实现，但**不是机械翻译**：

| 维度 | OpenHarness (Python) | JHarness (Java) |
|------|----------------------|-----------------|
| **语言** | Python 3.10+ | Java 17+ |
| **打包** | pip install | 单 fat jar (`mvn package`) |
| **HTTP / SSE** | httpx | OkHttp + `okhttp-sse` |
| **CLI 框架** | Typer / Click | Picocli |
| **TUI** | Textual | Lanterna |
| **校验** | pydantic | Jakarta Validation + Hibernate Validator |
| **包结构** | flat 风格 | 分层（kernel / capability / integration / extension / tools / command） |
| **SPI 隔离** | 弱（duck typing） | 强（`LlmGateway` / `ToolCatalog` / `PermissionGate` 显式接口） |
| **依赖治理** | requirements.txt | Maven Enforcer + ArchUnit 双守卫 |
| **Plugin manifest** | 自定义 | 兼容 Claude Code `plugin.json` 字段 |

JHarness 的工具集、斜杠命令、Hook 事件、MCP 协议都做了对齐，**插件可以在两边生态间互通**（只要不依赖语言特性的脚本）。

## 3. 特性矩阵

### 3.1 核心能力

| 类别 | 能力 |
|------|------|
| **查询引擎** | ReAct 循环 · 多工具并行 · SSE 流式 · 中途 `cancel()` · 消息压缩 · 成本追踪 · `maxTurns` 防失控 |
| **LLM Provider** | OpenAI 兼容协议（OpenAI / DeepSeek / Qwen / Kimi / DashScope / vLLM / Ollama 等） · 自动重试（指数退避） · 本地端点免 Key 占位 |
| **权限系统** | 3 模式 × 责任链评估 × 工具白/黑名单 × 路径 glob 规则 × 命令模式黑名单 × 审计日志 |
| **会话** | JSON 快照持久化 · `--resume <id>` / `--continue` 恢复 · 自动保存 · 压缩 · 回滚（`/rewind`） |
| **记忆** | 按项目分类 · 关键词搜索 · 过期清理 · 跨会话持久化 |

### 3.2 扩展生态

| 类别 | 能力 |
|------|------|
| **工具** | 40+ 内置工具 · JSON Schema 自动生成（基于 Jakarta Validation 注解反射） · 只读 / 写入语义自描述 |
| **斜杠命令** | 60+ 命令 · 三种来源（内置 / 插件 `commandsDir` / Markdown 技能） |
| **插件** | `~/.jharness/plugins/` + `<project>/.jharness/plugins/` · `plugin.json` 兼容 Claude Code · 信任签名（`TrustStore`） · 插件市场源 |
| **技能** | 三级加载（内置 → 用户 → 项目） · Markdown + YAML frontmatter · 自动注入系统提示词 |
| **Hook** | 8 类事件 × 5 类 Runner（Command / HTTP / Prompt / Agent / JavaSubprocess） · 递归深度防护 |
| **MCP** | stdio + HTTP 双传输 · JSON-RPC 2.0 · 动态工具补注册 · stdio fork 前权限评估 |
| **多 Agent** | `executeParallel` / `executeSequential` / `executeSingle` · Team 角色管理 · 命名线程池 + 回压 |
| **后台任务** | Shell / Agent 后台任务 · 进程隔离 · 输出落盘 · `task_*` 工具族 |
| **Cron** | 持久化定时任务 · `ScheduledExecutorService` · `remote_trigger` 按需触发 |

### 3.3 工程化

| 类别 | 能力 |
|------|------|
| **依赖治理** | Maven Enforcer：Java 17+ / Maven 3.6.3+ / `dependencyConvergence` / 禁止重复版本 |
| **架构守卫** | ArchUnit 单测固化分层依赖方向（kernel 不依赖 capability/integration/tools） |
| **代码格式** | Spotless + Google Java Format + 自动移除未用 import |
| **测试** | JUnit 5 + Mockito + MockWebServer + ArchUnit |
| **打包** | maven-shade-plugin fat jar，主类 `JHarnessApplication` |

## 4. 使用形态

### 4.1 CLI 模式

```bash
# 交互式
java -jar jharness.jar

# 单次查询
java -jar jharness.jar -p "总结一下这个项目"

# 恢复会话
java -jar jharness.jar -r abc12345
```

### 4.2 TUI 模式

```bash
java -jar jharness.jar --tui
```

基于 Lanterna 的全屏终端 UI，支持 Markdown 渲染、状态栏、流式打字效果。

### 4.3 嵌入式（Library）

```java
Settings settings = Settings.load();
QueryEngine engine = new QueryEngineBuilder(settings, "default", null).build();
engine.submitMessage("你好", event -> System.out.print(event)).join();
```

把 JHarness 当作普通 Java 库引入，自己控制 UI、消息持久化、Hook 编排。

## 5. 适用场景

✅ **个人开发者** — 在终端里用本地 Ollama 跑一个隐私安全的编程助手
✅ **企业 IDE 插件** — 把 Agent 内核嵌入到 IDEA Plugin / VSCode JVM 后端
✅ **DevOps 平台** — Cron + Hook + Shell 工具组合，做"AI 运维巡检"
✅ **多 Agent 编排** — 用 `AgentOrchestrator` 做"产品 → 架构 → 编码 → 评审"流水线
✅ **MCP 网关** — 把 MCP 生态服务（GitHub / Filesystem / Postgres）接入 JVM 系统

❌ **超大上下文窗口需求** — 默认按 token 预算压缩，超长上下文建议自行扩展 `MessageCompactionService`
❌ **GUI 桌面应用** — 框架本身只提供 CLI + TUI，桌面端需自己包装
