# 13 · 多智能体协调

> 包路径：`io.leavesfly.jharness.capability.coordination`
>
> 核心类：`AgentOrchestrator`（编排器） + `TeamRegistry` + `TeamRecord`

---

## 1. 概念

多 Agent 协调允许在一个 JHarness 进程内运行多个子 Agent，按以下三种模式协作：

- **并行（parallel）** — 多个独立任务同时执行，结果聚合
- **顺序（sequential）** — 串行执行，前一个的输出可作为后一个的输入
- **单任务（single）** — 一次性派发一个任务，等待结果

每个子 Agent 拥有**独立的 `QueryEngine` 实例**（独立消息历史、独立系统提示词），但共用：

- 同一个 `OpenAiApiClient`（或子类）
- 同一个 `ToolCatalog`（工具目录 SPI）
- 同一个 `PermissionGate`（权限闸门）

这保证了"主 Agent + N 个子 Agent"运行在同一进程内，避免 fork JVM 的开销。

## 2. 核心 API：`AgentOrchestrator`

```java
public class AgentOrchestrator {
    public AgentOrchestrator(QueryEngine queryEngine, ToolCatalog toolRegistry);

    public void setQueryEngineFactory(QueryEngineFactory factory);
    public void setHookEmitter(Object hookExecutor);

    public CompletableFuture<List<AgentResult>> executeParallel(
        List<AgentTask> tasks, Consumer<StreamEvent> eventConsumer);

    public CompletableFuture<List<AgentResult>> executeSequential(
        List<AgentTask> tasks, Consumer<StreamEvent> eventConsumer);

    public CompletableFuture<AgentResult> executeSingle(
        AgentTask task, Consumer<StreamEvent> eventConsumer);

    public List<AgentInstance> getActiveAgents();
    public AgentInstance getAgentStatus(String agentId);
    public void cleanupCompletedAgents();
    public void shutdown();
}
```

### 2.1 工厂模式

```java
@FunctionalInterface
public interface QueryEngineFactory {
    QueryEngine create(String model, String systemPrompt);
}
```

工厂由 `QueryEngineBuilder` 提供，每个 `AgentTask` 创建独立的 `QueryEngine` 实例：

```java
orchestrator.setQueryEngineFactory((model, sysPrompt) -> {
    return new QueryEngine(apiClient, toolRegistry, permissionChecker,
                           cwd, sysPrompt, maxTurns);
});
```

public static class AgentTask {

## 4. 三种执行模式

### 4.1 并行执行（`executeParallel`）

```java
List<AgentTask> tasks = List.of(
    new AgentTask("t1", "frontend-review", "审查前端代码", null, frontendPrompt),
    new AgentTask("t2", "backend-review",  "审查后端代码", null, backendPrompt),
    new AgentTask("t3", "test-review",     "审查测试代码", null, testPrompt)
);

CompletableFuture<List<AgentResult>> future = orchestrator.executeParallel(tasks, event -> {
    // 流式事件实时打印（来自任一子 Agent）
});

List<AgentResult> results = future.join();
```

**线程池配置**：

```java
int cores = Runtime.getRuntime().availableProcessors();
int maxPool = Math.max(8, cores * 2);
new ThreadPoolExecutor(
    Math.min(4, maxPool),              // corePool
    maxPool,                            // maxPool
    60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(128),     // 有界队列
    namedThreadFactory("jharness-agent-"),
    new CallerRunsPolicy()              // 满队列时回压调用线程，避免静默丢任务
);
```

- 有界线程池：避免 `newCachedThreadPool` 无界扩展引发 OOM
- 有界队列 + CallerRuns：队列满时由调用线程亲自跑任务，给调用方明确的回压信号

### 4.2 顺序执行（`executeSequential`）

```java
List<AgentTask> tasks = List.of(
    new AgentTask("t1", "design",  "设计架构方案", null, designPrompt),
    new AgentTask("t2", "review",  "审查设计方案", null, reviewPrompt),  // 接收 t1 输出
    new AgentTask("t3", "refine",  "根据审查意见优化", null, refinePrompt)
);

CompletableFuture<List<AgentResult>> future = orchestrator.executeSequential(tasks, ec);
```

内部实现（`AgentOrchestrator#executeSequential`）：单线程串行 `for` 循环，**前一个任务的 `output` 自动拼接到下一个任务的 prompt 前面**，拼接模板固定为：

```
前序任务结果:
<上一个任务的 output>

<再上一个的 output>

当前任务:
<本任务原 prompt>
```

只有 `result.isSuccess()` 的输出才会进入 `previousResults` 列表；失败任务的输出不会污染下游。所有累计的成功输出会按顺序全部拼到下一个任务的"前序任务结果"段中，**没有 token 截断**，调用方需要自己控制任务链长度避免上下文溢出。

### 4.3 单任务（`executeSingle`）

最常用，被 `agent_spawn` 工具直接调用：

```java
AgentTask task = new AgentTask("t1", "research", "调研 X 技术", model, sysPrompt);
AgentResult result = orchestrator.executeSingle(task, ec).join();
```

执行结束后自动发射 `SUBAGENT_STOP` Hook 事件（如已注入 `hookEmitter`）。

## 5. 子代理定义（来自插件 / 用户配置）

子代理的"角色定义"（系统提示词）可以来自三处：

| 来源 | 位置 |
|------|------|
| **插件** | `<plugin>/agents/<name>.md`（YAML frontmatter + Markdown 正文） |
| **项目** | `<cwd>/.jharness/agents/<name>.md` |
| **用户** | `~/.jharness/agents/<name>.md` |

加载后存进 `TeamRegistry` 或 `SkillRegistry`，供 `agent_spawn` 工具按 `name` 查找。

示例 `agents/code-reviewer.md`：

```markdown
---
name: code-reviewer
description: 严格的代码审查员
---

你是一位严格的资深 Java 工程师，专注于：
- 代码可读性
- 并发安全
- 性能瓶颈
- 单元测试覆盖

回答时请逐条列出问题，并给出具体修改建议。
```

## 6. `agent_spawn` 工具

供 LLM 主动派发子任务：

```json
{
  "tool_calls": [{
    "function": {
      "name": "agent_spawn",
      "arguments": "{\"name\":\"code-reviewer\",\"task\":\"审查 src/Main.java\"}"
    }
  }]
}
```

工具实现：

1. 从 `TeamRegistry` / `SkillRegistry` 查 `name` → 找到 `systemPrompt`
2. 构造 `AgentTask`
3. 调 `orchestrator.executeSingle(task, ec)`
4. 把 `AgentResult.output` 作为 `ToolResult` 返回

LLM 在下一轮看到工具结果，可以进一步引用 / 综合。

## 7. 团队管理（`TeamRegistry` + `TeamRecord`）

`TeamRegistry` 让多个相关任务被归入"团队"，便于跟踪：

### 7.1 创建团队

```java
TeamRegistry teams = new TeamRegistry(orchestrator);

TeamRecord team = teams.createTeam("backend-refactor", "后端重构小组");
team.addAgent("task-1", "designer");
team.addAgent("task-2", "reviewer");
team.addMessage("开始", "system");
team.setStatus("active");
```

### 7.2 `TeamRecord` 数据模型

```java
public class TeamRecord {
    String id;               // 完整 UUID（避免截断碰撞）
    String name;             // 团队名（同时是 key）
    String description;
    Instant createdAt;
    List<AgentInfo> agents;        // CopyOnWriteArrayList
    List<TeamMessage> messages;
    Map<String, String> metadata;
    String status;           // active / deleted / paused
}
```

`AgentInfo` 含 `taskId` / `role` / `addedAt`；`TeamMessage` 含 `content` / `sender` / `timestamp`。

### 7.3 `/agents` 命令

```
/agents teams           列出所有团队
/agents team <name>     查看团队详情
/agents create <name>   创建团队
/agents delete <name>   删除团队
/agents list            列出所有运行中的 agent
/agents status <id>     查看 agent 状态
/agents stop <id>       停止
```

由 `AgentCommandHandler` 实现。

## 8. Hook 集成

`AgentOrchestrator.setHookEmitter(hookExecutor)` 注入 `HookExecutor` 实例后：

- `executeSingle` 结束时反射调用 `hookExecutor.execute(SUBAGENT_STOP, payload)`
- payload 包含 `agent_id` / `task_name` / `success` / `error`（如有）

反射调用避免 `capability.coordination` 强依赖 `capability.hook`，符合分层架构。

## 9. 子 Agent 通信

`send_message` 工具允许向运行中的子 Agent 发消息：

```json
{
  "tool_calls": [{
    "function": {
      "name": "send_message",
      "arguments": "{\"agent_id\":\"t1\",\"message\":\"补充信息：xxx\"}"
    }
  }]
}
```

内部调用 `BackgroundTaskManager.writeToTask(agentId, message)`（仅对 LOCAL_AGENT 类型的子进程 Agent 有效）。

## 11. 完整示例：并行审查多个文件

```java
// 1. 初始化
AgentOrchestrator orchestrator = new AgentOrchestrator(mainEngine, toolRegistry);
orchestrator.setQueryEngineFactory((model, sys) -> queryEngineBuilder.buildChild(model, sys));
orchestrator.setHookEmitter(hookExecutor);

// 2. 准备任务
String reviewerPrompt = "你是资深 Java 工程师，请审查代码并指出问题。";
List<String> files = List.of("src/Main.java", "src/Util.java", "src/Service.java");

List<AgentOrchestrator.AgentTask> tasks = files.stream()
    .map(f -> new AgentOrchestrator.AgentTask(
        "review-" + f, "review-" + f,
        "请审查文件: " + f, null, reviewerPrompt))
    .toList();

// 3. 并行执行
List<AgentOrchestrator.AgentResult> results = orchestrator
    .executeParallel(tasks, event -> { /* 实时打印 */ })
    .join();

// 4. 汇总
for (var r : results) {
    System.out.println("[" + r.getTaskName() + "] " + (r.isSuccess() ? "OK" : "FAIL"));
    System.out.println(r.getOutput());
}

// 5. 关闭
orchestrator.shutdown();
```

## 12. 与后台任务的区别

| | `BackgroundTaskManager`（task 包） | `AgentOrchestrator`（coordination 包） |
|---|-----------------------------------|---------------------------------------|
| **任务类型** | Shell 命令 / 子进程 Agent | 进程内 QueryEngine |
| **隔离级别** | 进程级 | 线程级 |
| **资源开销** | 高（fork 进程） | 低（共享 JVM） |
| **流式事件** | 通过文件 / 管道 | 通过 `Consumer<StreamEvent>` |
| **典型场景** | 长运行命令、独立 Agent | 短任务并行 / 子代理 |

详见 [14-后台任务与Cron](14-后台任务与Cron.md)。
