# 07 · 斜杠命令系统

> 包路径：`io.leavesfly.jharness.command`
>
> 核心类：`SlashCommand` 接口 + `CommandRegistry` + `SimpleSlashCommand` + `PluginSlashCommand`

---

## 1. 概念

**斜杠命令** = 用户在交互式 REPL 中输入 `/<name> <args...>` 触发的本地命令处理器。

斜杠命令与工具的关键区别：

| | 斜杠命令 | 工具 |
|---|---------|------|
| **触发者** | 用户 | LLM |
| **执行地点** | 本地 JVM | 本地 JVM |
| **输入** | `List<String> args` | 强类型 DTO |
| **输出** | `CommandResult`（status + message） | `ToolResult`（content + metadata） |
| **进入消息历史** | ❌（除非命令显式 submit） | ✅（`ToolResultBlock`） |
| **典型用途** | 切换配置、查看状态、辅助操作 | LLM 完成任务的"手脚" |

---

## 2. 核心接口

### 2.1 `SlashCommand`

```java
public interface SlashCommand {
    String getName();
    String getDescription();
    CompletableFuture<CommandResult> execute(
        List<String> args, CommandContext context, Consumer<StreamEvent> eventConsumer);
}
```

### 2.2 `CommandContext`

```java
public class CommandContext {
    private final Path cwd;                       // 当前工作目录
    private final QueryEngine engine;             // 引擎实例（可 null）
    private final ToolRegistry toolRegistry;      // 工具注册表
    private final PermissionGate permissionChecker;
    private final Settings settings;
    private final AppStateStore appStateStore;    // 应用运行时状态
}
```

### 2.3 `CommandResult`

```java
public class CommandResult {
    private final boolean success;
    private final String message;

    public static CommandResult success(String message);
    public static CommandResult error(String message);
}
```

## 3. 三种实现方式

### 3.1 `SimpleSlashCommand`（lambda 风格）

适合无状态的简单命令：

```java
register(new SimpleSlashCommand("greet", "打个招呼",
    (args, ctx, ec) -> CompletableFuture.completedFuture(
        CommandResult.success("Hello, " + (args.isEmpty() ? "world" : args.get(0))))));
```

`CommandRegistry` 内部用 `cmd(name, desc, handler)` 静态工厂简写。

### 3.2 完整类（推荐用于复杂命令）

参考 `command/builtin/system/MemoryCommand.java`、`config/ConfigCommandHandler.java`：

```java
public class MemoryCommand {
    public static SlashCommand create(MemoryManager mgr) {
        return new SimpleSlashCommand("memory", "记忆管理", (args, ctx, ec) -> {
            // 多子命令分发
            String sub = args.isEmpty() ? "list" : args.get(0);
            return switch (sub) {
                case "add" -> doAdd(mgr, args);
                case "search" -> doSearch(mgr, args);
                case "list" -> doList(mgr);
                default -> CompletableFuture.completedFuture(
                    CommandResult.error("未知子命令: " + sub));
            };
        });
    }
}
```

### 3.3 `PluginSlashCommand`（插件来源）

由插件的 `commands/*.md` 注册而成。把 Markdown 正文作为 prompt 模板，用户参数按占位符注入后作为新的用户消息提交给 `QueryEngine`：

```java
PluginSlashCommand cmd = PluginSlashCommand.fromSkill(skillDefinition, "myplugin");
```

**占位符规则**（对齐 Claude Code）：

| 占位符 | 替换为 |
|--------|--------|
| `$ARGUMENTS` | 全部 args 用空格连接 |
| `{{args}}` | 同上 |
| `$1`, `$2`, ..., `$99` | 第 i 个 arg（越界为空串） |

实现注意：`$1` 用正则一次性替换，避免"`$1` 先于 `$10` 被替换"导致字符串腐蚀（旧 bug 已修）。

## 4. 注册流程

### 4.1 基础注册

```java
CommandRegistry registry = new CommandRegistry();   // 构造时自动调 registerAllCommands()
```

`registerAllCommands()` 注册全部内置命令（详见 § 5）。

### 4.2 完整注册（带 Manager 依赖的命令）

```java
CommandRegistry registry = CommandRegistry.createFullRegistry(
    settings, queryEngine, toolRegistry, mcpManager,
    teamRegistry, orchestrator, cronRegistry, hookRegistry);
```

会补注册：

- `/mcp` — 基于 `McpClientManager` 的完整管理（覆盖基础版的简易实现）
- `/agents` — `AgentCommandHandler`
- `/cron` — `CronCommandHandler`（如果未注册过）
- `/hooks` — `HooksCommand.createWithRegistry`（支持持久化 / 触发）

幂等：基础版已注册的命令不会被覆盖（`hasCommand` 检查）。

### 4.3 命令查找

```java
Optional<SlashCommand> cmd = registry.lookup("/memory add 项目 标题 内容");
// 自动去掉前导 "/"
// 截取空格前的 name 部分（"memory"）
// 大小写不敏感
```

## 6. 按键绑定

`command/keybindings/` 子包提供 KeyStroke → 命令名 的映射，UI 层在收到对应按键时执行命令。

### 6.1 默认绑定

| 按键 | 命令 |
|------|------|
| `Ctrl+L` | `clear` |
| `Ctrl+K` | `toggle_vim` |
| `Ctrl+V` | `toggle_voice` |
| `Ctrl+T` | `tasks` |

定义在 `DefaultKeybindings.getDefaults()`。

### 6.2 自定义

在 `~/.jharness/keybindings.json` 写入：

```json
{
  "ctrl+r": "resume",
  "ctrl+e": "export",
  "alt+enter": "newline"
}
```

由 `KeybindingLoader` 加载，`KeybindingParser` 反序列化为 `Map<String, String>`（仅支持字符串值，非字符串值在 Jackson 层就会失败）。

`KeybindingResolver` 把 `KeyStroke` 翻译为命令名（如 `ctrl+l`）。

## 7. 编写一个自定义命令

```java
// 简单版
SlashCommand cmd = new SimpleSlashCommand(
    "wc",
    "统计当前消息历史的字数",
    (args, ctx, ec) -> {
        int total = ctx.getEngine().getMessages().stream()
            .flatMap(m -> m.getContent().stream())
            .filter(b -> b instanceof TextBlock)
            .mapToInt(b -> ((TextBlock) b).getText().length())
            .sum();
        return CompletableFuture.completedFuture(
            CommandResult.success("消息历史共 " + total + " 字"));
    }
);

registry.register(cmd);
```

更复杂的可继承 `SlashCommand` 实现完整类，或参考 `command/builtin/system/CostCommand.java`。

## 8. 命令在 UI 中的接入

`ConsoleInteractiveSession` 在用户输入以 `/` 开头时：

1. `registry.lookup(input)` 找到命令
2. 解析剩余 args（按空格分词）
3. 调用 `cmd.execute(args, context, eventConsumer)`
4. 把 `CommandResult.message` 渲染到终端
5. 特殊返回值 `"exit"` 触发 REPL 退出

`TerminalUI`（Lanterna 模式）类似，额外处理按键绑定 → 命令分发。
