# 03 · 整体架构

> JHarness 采用**分层 + SPI 隔离**的架构：把"易变实现"和"稳定契约"显式拆开，让内核可以被替换、被测试、被嵌入。

---

## 1. 分层总览

```
┌──────────────────────────────────────────────────────────────┐
```

## 2. 关键设计原则

### 2.1 SPI 反向依赖隔离

**问题**：如果 `kernel.engine.QueryEngine` 直接 import `integration.api.OpenAiApiClient` / `capability.permission.PermissionChecker`，那么"内核"会被"实现"反向绑死，无法做单测、无法替换 Provider、无法被嵌入到其他 UI 中。

**方案**：在 `kernel.spi` 包定义 SPI 接口，内核只依赖接口；具体实现在 `integration` / `capability` 包，由 `app.bootstrap.QueryEngineBuilder` 在装配时注入。

```
kernel.spi.LlmGateway              ← QueryEngine 依赖
   ↑ extends
integration.api.LlmProvider        ← OpenAiApiClient 实现
```

| SPI 接口 | 作用 | 实现 |
|----------|------|------|
| `LlmGateway` | LLM 流式 / 取消 / Provider 元信息 | `integration.api.OpenAiApiClient` |
| `ToolCatalog` | 列出所有可用工具 | `tools.ToolRegistry` |
| `PermissionGate` | 评估工具是否被允许执行 | `capability.permission.PermissionChecker` |

`kernel.engine.QueryEngine` 整个签名只引用 SPI 接口和模型类（`ConversationMessage` / `StreamEvent`），**不 import 任何 integration / capability / tools 的具体实现**。

### 2.2 装配集中（Bootstrap 模式）

把"new XxxClient / new XxxRegistry / 注入 setter"等繁琐胶水代码集中到 `app.bootstrap.QueryEngineBuilder`：

```java
QueryEngine engine = new QueryEngineBuilder(settings, permissionMode, resumeSessionId).build();
```

这一行内部完成：

1. 构造 `OpenAiApiClient`
2. 构造 `BackgroundTaskManager` / `TeamRegistry` / `SkillRegistry` / `McpClientManager` / `CronRegistry`
3. 构造 `PermissionChecker` 并装配白/黑名单 / 路径规则 / 命令黑名单
4. `PluginBootstrap` 扫描插件、注册 plugin Agents、收集 hooks
5. `McpBootstrap` 注册 MCP 服务器、加载 `CLAUDE.md`
6. `ToolRegistry.withDefaults(...)` 注册 40+ 内置工具
7. 构造 `SystemPromptBuilder` 并注入技能元信息
8. new `QueryEngine`，配置 `CostTracker` / `MessageCompactionService` / `SessionPersister` / `HookExecutor`
9. 注册 close hook（释放 MCP / Cron / Task）
10. 配置 `AgentTool` 的 in-process 模式（让 `agent_spawn` 复用同一个 ApiClient）

`JHarnessApplication` 本身只剩 CLI 参数解析 + 调用 builder + 调用 `CliRunners`，**没有任何业务逻辑**。

### 2.3 接口 / 抽象类对称

- **抽象类** → 用于有共同字段或共同模板方法的扩展点（`BaseTool<T>` 提供 `toApiSchema` 反射、`OBJECT_MAPPER`）
- **接口** → 用于无状态的契约（`SlashCommand` / `LlmGateway` / `PermissionRule`）

### 2.4 线程模型

| 池 | 线程名前缀 | 默认大小 | 用途 |
|----|-----------|----------|------|
| `OpenAiHttpClient` OkHttp dispatcher | 默认 | 默认 | SSE 流式连接 |
| `BackgroundTaskManager` | `jharness-bg-task-N` | 20 (fixed) | 后台 Shell / Agent 子进程监管 |
| `AgentOrchestrator` | `jharness-agent-N` | core=4, max=2×CPU | 多 Agent 并行 / 顺序执行 |
| `CronRegistry.scheduler` | `jharness-cron-N` | 2 (scheduled) | Cron 任务调度（daemon） |
| `McpClientManager.executor` | `jharness-mcp-N` | `McpExecutorFactory.newDefault()` | MCP JSON-RPC IO |

所有池都用**命名 ThreadFactory + 有界队列 + 未捕获异常 handler**，便于 `jstack` 定位、防 OOM。

## 3. 模块依赖方向

允许的依赖方向（↓ 表示"上面 → 下面"）：

```
JHarnessApplication / app.cli
     ↓
app.bootstrap
     ↓                  ↘
ui.*                kernel.engine
                          ↓
                    kernel.spi      kernel.{memory,edit,plan,state}
                          ↑
        ┌─────────────────┼─────────────────┐
   capability.*      extension.*       integration.*
        ↓                                    ↓
   tools.* / command.* / config.* / prompt.* / util.*
```

**严禁的反向依赖**：

- ❌ `kernel.*` → `capability.*` / `integration.*` / `tools.*`
- ❌ `kernel.spi.*` → `integration.*` / `capability.*`
- ❌ `integration.*` → `capability.*`（已通过 `PermissionGate` SPI 解除）

这些规则用 **ArchUnit 单元测试**固化（依赖于 `com.tngtech.archunit:archunit-junit5`），任何破坏分层方向的代码改动都会在 `mvn test` 阶段失败。

## 4. 依赖治理

`pom.xml` 中启用了 Maven Enforcer Plugin：

```xml
<requireJavaVersion><version>[17,)</version></requireJavaVersion>
<requireMavenVersion><version>[3.6.3,)</version></requireMavenVersion>
<dependencyConvergence/>
<banDuplicatePomDependencyVersions/>
```

- `dependencyConvergence` 强制传递依赖只能解析出一个版本（曾因 `okhttp 4.12.0` 引入 `kotlin-stdlib` 1.8.21 / 1.9.10 分裂触发过失败，已通过 `dependencyManagement` 锁版本到 1.9.10 解决）
- `banDuplicatePomDependencyVersions` 禁止同一个 GAV 在 pom 中出现多次

## 6. 与上游 OpenHarness 的架构差异

| 维度 | OpenHarness | JHarness |
|------|-------------|----------|
| 模块隔离 | 一个大目录 | 分层包 + ArchUnit 守卫 |
| Provider 抽象 | 函数式 | SPI 接口 `LlmGateway` |
| 配置加载 | dict 直接传 | 强类型 `Settings` Bean + Jakarta Validation |
| Hook 实现 | 动态注册 callback | `HookRegistry` + `HookEvent` enum + 5 类 `HookRunner` |
| 线程池 | asyncio / TaskGroup | 显式命名 ThreadPool + `CallerRunsPolicy` |
| 工具 Schema | pydantic auto | `BaseTool#toApiSchema` 反射 + Jakarta Validation |
