# 06 · 工具系统

> 包路径：`io.leavesfly.jharness.tools`
>
> 核心类：`BaseTool<T>`（抽象基类） + `ToolRegistry`（注册表） + `ToolResult` + `ToolExecutionContext`

---

## 1. 概念

**工具**是 Agent 可调用的"外部能力单元"。每个工具：

- 有一个唯一的 `name`（snake_case）
- 有面向 LLM 的 `description`（用于决策何时调用）
- 接受强类型的输入 DTO（用 Jakarta Validation 注解描述约束）
- 异步执行并返回 `ToolResult`
- 自描述 `isReadOnly(input)`，供权限系统判断

LLM 通过 OpenAI `tool_calls` 协议发起调用，引擎从 `ToolRegistry` 拿到工具实例后并行执行。

---

## 2. `BaseTool<T>` 抽象基类

```java
public abstract class BaseTool<T> {
    protected static final ObjectMapper OBJECT_MAPPER = JacksonUtils.MAPPER;

    public abstract String getName();
    public abstract String getDescription();
    public abstract Class<T> getInputClass();
    public abstract CompletableFuture<ToolResult> execute(T input, ToolExecutionContext context);

    public boolean isReadOnly(T input) { return false; }

    public Map<String, Object> toApiSchema() { /* 反射生成 JSON Schema */ }
}
```

| 方法 | 说明 |
|------|------|
| `getName()` | 工具唯一名称，作为 `tool_calls` 的 `function.name` |
| `getDescription()` | LLM 决策依据，写清楚"何时用它 / 不要在何时用" |
| `getInputClass()` | 输入 DTO 的 Class，用于反序列化 + Schema 生成 |
| `execute(input, ctx)` | **必须返回 `CompletableFuture`**，长任务异步执行 |
| `isReadOnly(input)` | `DEFAULT` 模式下只读工具自动放行，写工具需要确认；`PLAN` 模式下写工具直接被拦截 |
| `toApiSchema()` | 默认实现：用 Jakarta Validation 注解反射生成 OpenAPI JSON Schema |

## 3. `ToolResult`

```java
public class ToolResult {
    private final boolean success;
    private final String content;
    private final Map<String, Object> metadata;

    public static ToolResult success(String content);
    public static ToolResult success(String content, Map<String, Object> metadata);
    public static ToolResult error(String message);
}
```

`content` 是返回给 LLM 的文本（会作为 `ToolResultBlock.content` 写回历史），`metadata` 仅作日志/UI 展示，不进入对话上下文。

---

## 4. `ToolExecutionContext`

```java
public class ToolExecutionContext {
    private final Path cwd;                              // 当前工作目录
    private final PermissionGate permissionChecker;      // 权限闸门
    // 其他扩展字段
}
```

工具内部需要工作目录、权限校验、或其他运行时上下文时通过这个对象拿到。

## 5. JSON Schema 自动生成

`BaseTool#toApiSchema()` 通过反射 + Jakarta Validation 注解自动生成 OpenAPI JSON Schema：

### 5.1 输入 DTO 示例

```java
public class FileReadToolInput {
    @NotBlank
    public String file_path;

    public Integer offset;
    public Integer limit;
}
```

### 5.2 生成的 Schema

```json
{
  "type": "object",
  "properties": {
    "file_path": { "type": "string" },
    "offset":    { "type": "integer" },
    "limit":     { "type": "integer" }
  },
  "required": ["file_path"]
}
```

### 5.3 类型映射规则

| Java 类型 | JSON Schema |
|-----------|-------------|
| `String` | `string` |
| `int/Integer/long/Long` | `integer` |
| `double/Double/float/Float` | `number` |
| `boolean/Boolean` | `boolean` |
| 数组 / `List<T>` | `array` + `items` |
| 其他对象 | `object` |

### 5.4 必填字段识别

字段标注以下任一注解即视为必填：

- `@NotBlank`
- `@NotNull`
- `@NotEmpty`

## 6. `ToolRegistry`

```java
public class ToolRegistry implements ToolCatalog {
    private final Map<String, BaseTool<?>> tools = new ConcurrentHashMap<>();

    public void register(BaseTool<?> tool);          // 重复名警告 + 保留先注册者
    public BaseTool<?> get(String name);
    public boolean has(String name);
    public Set<String> getToolNames();
    public Collection<BaseTool<?>> getAllTools();
    public List<Map<String, Object>> toApiSchema();  // 给 LLM 的 tools 数组
    public boolean unregister(String name);
    public int size();
    public int refreshMcpTools(McpClientManager mcp); // 异步补注册 MCP 动态工具
}
```

### 6.1 重复注册保护

```java
public void register(BaseTool<?> tool) {
    BaseTool<?> existing = tools.putIfAbsent(name, tool);
    if (existing != null) {
        logger.warn("工具名冲突，忽略后注册者: name={}, keep={}, skipped={}", ...);
    }
}
```

避免插件覆盖内置工具或两个插件相互覆盖。

### 6.2 一键注册全部内置工具

```java
ToolRegistry registry = ToolRegistry.withDefaults(
    settings, taskManager, teamRegistry, skillRegistry, mcpManager, cronRegistry);
```

注册的工具数取决于传入的 manager 是否为 null（MCP / Cron / Skill 都做了空判，便于精简部署）。

### 6.3 MCP 动态工具补注册

`McpClientManager.connectAll()` 是异步的，`withDefaults` 时 MCP 工具列表可能还是空。`refreshMcpTools(mcp)` 用 `putIfAbsent` 幂等补齐：

```java
mcpManager.onConnected(() -> {
    int added = toolRegistry.refreshMcpTools(mcpManager);
    logger.info("MCP 连接完成后新增 {} 个动态工具", added);
});
```

## 8. 编写一个工具

完整示例：实现 `word_count` 工具。

### 8.1 输入 DTO

```java
// io.leavesfly.jharness.tools.input.misc.WordCountToolInput
package io.leavesfly.jharness.tools.input.misc;

import jakarta.validation.constraints.NotBlank;

public class WordCountToolInput {
    @NotBlank
    public String text;

    public boolean by_line = false;
}
```

### 8.2 工具实现

```java
// io.leavesfly.jharness.tools.builtin.misc.WordCountTool
package io.leavesfly.jharness.tools.builtin.misc;

import io.leavesfly.jharness.tools.*;
import io.leavesfly.jharness.tools.input.misc.WordCountToolInput;
import java.util.concurrent.CompletableFuture;

public class WordCountTool extends BaseTool<WordCountToolInput> {
    @Override public String getName() { return "word_count"; }
    @Override public String getDescription() {
        return "统计文本字数。by_line=true 时按行返回，false 时返回总字数。";
    }
    @Override public Class<WordCountToolInput> getInputClass() { return WordCountToolInput.class; }
    @Override public boolean isReadOnly(WordCountToolInput input) { return true; }

    @Override
    public CompletableFuture<ToolResult> execute(WordCountToolInput in, ToolExecutionContext ctx) {
        if (in.by_line) {
            String[] lines = in.text.split("\n");
            StringBuilder sb = new StringBuilder();
            for (int i = 0; i < lines.length; i++) {
                sb.append("第 ").append(i + 1).append(" 行: ").append(lines[i].length()).append("\n");
            }
            return CompletableFuture.completedFuture(ToolResult.success(sb.toString()));
        }
        return CompletableFuture.completedFuture(
            ToolResult.success("总字数: " + in.text.length()));
    }
}
```

### 8.3 注册

在 `ToolRegistry.withDefaults` 或自己的 builder 中：

```java
registry.register(new WordCountTool());
```

或在插件目录通过 `plugin.json` 声明。

### 8.4 LLM 调用

LLM 会自动生成：

```json
{
  "tool_calls": [{
    "id": "call_1",
    "type": "function",
    "function": {
      "name": "word_count",
      "arguments": "{\"text\":\"Hello World\",\"by_line\":false}"
    }
  }]
}
```

`ToolCallDispatcher` 反序列化 `arguments` → `WordCountToolInput` → 调用 `execute` → 把 `ToolResult.content` 写回 `ToolResultBlock`。

## 9. 工具并发与超时

引擎的 `ToolCallDispatcher`：

- 单个工具 → 直接 `.join()`，**无显式超时**（依赖工具内部超时）
- 多个工具 → `CompletableFuture.allOf(...).get(5, TimeUnit.MINUTES)`，超时全部返回 `ToolResult.error("工具执行超时")`

工具内部应自己控制长任务超时（如 `bash` 用 `Process.waitFor(timeout, unit)`，`web_fetch` 用 OkHttp readTimeout）。

## 10. 工具与权限

工具执行前会被 `PermissionChecker.evaluate(toolName, isReadOnly, filePath, command)` 评估：

| 决策 | 行为 |
|------|------|
| `allow()` | 直接执行 |
| `deny(reason)` | 返回 `ToolResult.error("权限被拒绝: " + reason)` |
| `requiresConfirmation(reason)` | UI 弹确认框，用户允许后才执行 |

工具实现应自己判断输入路径 / 命令，并通过 `toolName` + `filePath` + `command` 参数让权限规则识别。详见 [08-权限系统](08-权限系统.md)。
