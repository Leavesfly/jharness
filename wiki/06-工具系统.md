# 06 - 工具系统

> 位于 `io.leavesfly.jharness.tools`。工具系统是 Agent 的"手"，每个工具都是可被 LLM 调用的原子能力。

## 1. 核心抽象：`BaseTool<T>`

```java
public abstract class BaseTool<T> {
    protected static final ObjectMapper OBJECT_MAPPER = JacksonUtils.MAPPER;

    public abstract String getName();                 // 唯一工具名（LLM 识别）
    public abstract String getDescription();          // 供 LLM 理解的描述
    public abstract Class<T> getInputClass();         // 强类型输入
    public abstract CompletableFuture<ToolResult> execute(T input, ToolExecutionContext ctx);

    public boolean isReadOnly(T input) { return false; }   // 权限检查依据

    // 反射 + Jakarta Validation 自动生成 JSON Schema
    public Map<String, Object> toApiSchema() { ... }
}
```

**四条设计约定**：

1. **强类型输入**：`T` 通常是一个 POJO DTO，通过 Jackson 从 LLM 返回的 JSON 反序列化。
2. **异步执行**：返回 `CompletableFuture<ToolResult>`，允许并行调度。
3. **只读声明**：`isReadOnly(input)` 直接影响 `PermissionChecker` 决策。
4. **Schema 自动生成**：字段上的 `@NotBlank/@NotNull/@NotEmpty` 标注会映射到 `required` 列表。

## 2. `ToolResult` 返回结构

```java
public class ToolResult {
    boolean success;
    String content;       // 纯文本或 JSON 字符串
    String error;         // 失败原因

    static ToolResult success(String content) { ... }
    static ToolResult error(String error) { ... }
}
```

返回后会被包装成 `ToolResultBlock` 追加到对话历史，作为下一轮 LLM 的上下文。

## 3. `ToolExecutionContext` 上下文

```java
public class ToolExecutionContext {
    Path cwd;                             // 当前工作目录
    PermissionChecker permissionChecker;  // 深层工具可二次校验
    Consumer<StreamEvent> eventConsumer;  // 允许工具中途发送进度事件
}
```

## 4. `ToolRegistry`

### 4.1 注册 / 查询 API

```java
registry.register(new MyTool());        // 重复 name 会 putIfAbsent + warn 日志
registry.unregister("my_tool");
registry.get("my_tool");
registry.has("my_tool");
registry.getToolNames();
registry.getAllTools();
registry.size();
registry.toApiSchema();                 // 生成 OpenAI tools 字段
```

底层使用 `ConcurrentHashMap` + **快照读取** 策略 (`new ArrayList<>(tools.values())`)，保证并发安全且生成 Schema 过程不会抛 CME。

### 4.2 `withDefaults` 默认工具装配

```java
ToolRegistry.withDefaults(settings, taskManager, teamRegistry,
                          skillRegistry, mcpManager, cronRegistry)
```

一键注册以下工具（按类别）：

| 类别 | 工具 |
|------|------|
| **文件核心** | `FileReadTool`、`FileWriteTool`、`FileEditTool`、`MultiEditTool`、`UndoEditTool`、`GlobTool`、`GrepTool` |
| **代码智能** | `LspTool`（Java / Python LSP 客户端） |
| **Notebook** | `NotebookEditTool` |
| **系统** | `BashTool`、`SleepTool` |
| **任务管理** | `TodoWriteTool`、`TaskCreateTool`、`TaskGetTool`、`TaskListTool`、`TaskStopTool`、`TaskOutputTool`、`TaskUpdateTool` |
| **搜索 / 元** | `WebFetchTool`、`WebSearchTool`、`ToolSearchTool`、`BriefTool`、`ConfigTool` |
| **模式切换** | `EnterPlanModeTool`、`ExitPlanModeTool`、`EnterWorktreeTool`、`ExitWorktreeTool` |

| **Cron** | `CronCreateTool`、`CronListTool`、`CronDeleteTool`、`RemoteTriggerTool` |

## 5. 工具分类详解

### 5.1 文件操作

| 工具 | 关键能力 |
|------|---------|
| `FileReadTool` | 按行范围读取，支持 `start_line` / `end_line`，大文件防溢出 |
| `FileWriteTool` | 整文件写入，目录自动创建 |
| `FileEditTool` | 查找 + 替换，要求 `old_string` 必须唯一匹配 |
| `MultiEditTool` | 单文件多块编辑，原子性提交 |
| `UndoEditTool` | 基于 `EditHistoryManager` 回滚最近一次编辑 |
| `GlobTool` | glob 模式查找，支持 `**/*.java` 递归 |
| `GrepTool` | 文本 / 正则内容搜索 |

### 5.2 代码智能

- **`LspTool`（23 KB 实现）**：内置 LSP 客户端，子命令 `getSymbols` / `gotoDefinition` / `findReferences` / `getHover`，支持 Java（jdtls）、Python（pylsp）。
- **`NotebookEditTool`**：操作 `.ipynb`，支持按 `cellIndex` 创建 / 编辑 / 删除单元格，保留执行计数。

### 5.3 系统交互

- **`BashTool`（15 KB）**：
  - 强制 `bash -c` 执行，带工作目录、环境变量、超时（默认 120s，可配置）
  - 输出大小上限 2 MB（防日志爆炸）
  - 走 `PermissionChecker` 的命令黑名单校验
  - 支持 `is_background=true` 转交 `BackgroundTaskManager`
- **`SleepTool`**：`Thread.sleep(ms)`，主要用于调试或 ReAct 节奏控制。

### 5.4 任务管理

- **`TodoWriteTool`**：写入当前会话的 TODO list，被 Claude 式 Agent 用作"脑图"。
- **`TaskCreateTool` / `TaskGetTool` / `TaskListTool` / `TaskStopTool` / `TaskOutputTool` / `TaskUpdateTool`**：后台 shell 任务全生命周期，底层 `BackgroundTaskManager`。

### 5.5 多智能体

- **`AgentTool`（`agent_spawn`）**：
  - 配置 `configureInProcess(apiClient, toolRegistry, checker)` 后，使用主进程共享的 `QueryEngine` 工厂，无需启动子进程
  - 支持单任务和并行多任务
- **`SendMessageTool`**：对指定 agent 发送消息，`BackgroundTaskManager.writeToTask` 管道通信
- **`TeamCreateTool` / `TeamDeleteTool`**：`TeamRegistry` 操作

### 5.6 网络

- **`WebFetchTool`**：`OkHttp` 下载 + `Jsoup` 清洗 HTML → 纯文本，走 `UrlSafetyValidator` 防 SSRF。
- **`WebSearchTool`**：DuckDuckGo HTML 抓取，`Jsoup` 解析结果列表。

### 5.7 MCP

- **`ListMcpResourcesTool` / `ReadMcpResourceTool`**：暴露 MCP `resources/*` 协议。
- **`McpAuthTool`**：OAuth 流程（见 [12](12-MCP协议客户端.md)）。
- **`McpToolAdapter`**：运行时桥接 —— 把 MCP 服务器声明的 tool 包装成 `BaseTool`，执行时转发 `callTool` JSON-RPC。

### 5.8 Cron

- **`CronCreateTool`**：写入 `~/.jharness/data/cron_jobs.json` 并通过 `CronRegistry.upsertJob` 加入 `ScheduledExecutorService`。
- **`CronListTool` / `CronDeleteTool`**：管理作业。
- **`RemoteTriggerTool`**：按需即时触发作业（HTTP webhook 风格）。

### 5.9 模式切换

- **`EnterPlanModeTool` / `ExitPlanModeTool`**：切换 `settings.permissionMode` 及 `PermissionChecker.setMode`，与 `ExecutionPlan` 协同。
- **`EnterWorktreeTool` / `ExitWorktreeTool`**：Git worktree 隔离执行区。

### 5.10 元能力

- **`ToolSearchTool`**：在当前 `ToolRegistry` 中模糊搜索（`name` / `description`），支持 LLM 按需查询工具清单。
- **`BriefTool`**：生成当前上下文的摘要。
- **`ConfigTool`**：运行时 get/set 配置（仅限可修改字段）。
- **`AskUserQuestionTool`**：向用户抛出结构化问题（UI 侧渲染为选择框）。
- **`SkillTool`**：执行预定义技能。

## 6. Schema 自动生成机制

```java
class MyToolInput {
    @NotBlank  public String path;
    @NotNull   public Integer maxLines;
               public Boolean verbose;      // 非必填
               public List<String> tags;    // array of object
}
```

`toApiSchema()` 反射后生成：

```json
{
  "type": "object",
  "properties": {
    "path":     {"type": "string"},
    "maxLines": {"type": "integer"},
    "verbose":  {"type": "boolean"},
    "tags":     {"type": "array", "items": {"type": "object"}}
  },
  "required": ["path", "maxLines"]
}
```

类型映射表（`mapJavaTypeToJsonType`）：

| Java | JSON Schema |
|------|-------------|
| `String` | `string` |
| `int/Integer/long/Long` | `integer` |
| `double/Double/float/Float` | `number` |
| `boolean/Boolean` | `boolean` |
| `T[]` / `List<T>` | `array` |
| 其它 | `object` |

## 7. 权限集成

每次工具被调用前由 `QueryEngine#executeToolCall` 执行：

```
PermissionDecision d = permissionChecker.evaluate(
    tool.getName(),
    tool.isReadOnly(input),
    extractFilePath(input),     // 识别 file_path / path / filePath
    extractCommand(input));     // 识别 command / cmd
```

因此**工具实现者只需正确返回 `isReadOnly`**，其余交给权限层。

## 8. 自定义工具示例

```java
public class HelloTool extends BaseTool<HelloTool.Input> {

    public static class Input {
        @NotBlank public String name;
    }

    @Override public String getName() { return "hello"; }
    @Override public String getDescription() { return "向某人打招呼"; }
    @Override public Class<Input> getInputClass() { return Input.class; }
    @Override public boolean isReadOnly(Input input) { return true; }

    @Override
    public CompletableFuture<ToolResult> execute(Input input, ToolExecutionContext ctx) {
        return CompletableFuture.completedFuture(
            ToolResult.success("你好，" + input.name + "！"));
    }
}

// 注册
toolRegistry.register(new HelloTool());
```
